## Y Combinator Framework

YC's methodology is deeply integrated into this skill. See shared references for comprehensive YC content.

### Core YC Principles for Startups

| Principle | Application |
|-----------|-------------|
| **"Make Something People Want"** | Primary validation criterion |
| **Launch in 2 weeks** | MVP timeline target |
| **5-7% weekly growth** | Minimum healthy growth rate |
| **Do things that don't scale** | Manual tactics before automation |
| **Talk to users** | 10+ conversations/week minimum |

### YC MVP Strategy

> "If you can't build an MVP in 2 weeks, you're building too much." — Michael Seibel

**90/10 Solution:** Build 90% of the value with 10% of the effort. Focus ruthlessly on the core problem.

### MVaP: Minimum VALUABLE Product

Traditional MVP thinking leads to "Maximally Undifferentiated Products" (MUP). Shift your thinking:

| Concept | Traditional MVP | MVaP (Minimum Valuable Product) |
|---------|----------------|--------------------------------|
| **V means** | Viable (it works) | Valuable (they want it) |
| **Focus** | Building something | Building something DIFFERENT |
| **Features** | All equal priority | Delighters > Performance > Must-Haves |
| **Result** | "It works" | "I can't live without this" |

**The Feature Cocktail:**
1. **Delighter** (80% of innovation) - ONE feature that creates "wow"
2. **Performance** - Meet minimum threshold to compete
3. **Must-Haves** - Use existing solutions, don't reinvent

See: `../shared/references/mvap-framework.md` for complete framework.

### YC PMF Definition

> "You have product-market fit when you're drowning in demand." — Michael Seibel

**The Sean Ellis Test:** 40%+ of users would be "very disappointed" if product disappeared.
