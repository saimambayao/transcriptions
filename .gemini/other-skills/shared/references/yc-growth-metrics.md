## YC Growth Philosophy

> "A startup is a company designed to grow fast. Being newly founded does not in itself make a company a startup. Nor is it necessary for a startup to work on technology, or take venture funding, or have some sort of 'exit.' The only essential thing is growth." — Paul Graham

## Weekly Growth Benchmarks

### YC Growth Rate Targets

| Weekly Growth | Classification | Annual Result |
|---------------|----------------|---------------|
| **1-2%** | Struggling | 1.7x - 2.8x |
| **3-4%** | Surviving | 4.8x - 7.7x |
| **5-7%** | Good | 12.6x - 33.7x |
| **10%** | Exceptional | 142x |

### The Power of Compound Growth

```
5% weekly:  Year 1 = 12.6x,  Year 2 = 159x
7% weekly:  Year 1 = 33.7x,  Year 2 = 1,135x
10% weekly: Year 1 = 142x,   Year 2 = 20,000x+
```

**YC Insight:** Small differences in weekly growth create enormous differences over time.

## Growth Stages

### Stage-Appropriate Metrics

| Stage | Primary Metric | Target |
|-------|---------------|--------|
| **Pre-launch** | User interviews | 10+/week |
| **MVP** | Active users | Week-over-week increase |
| **PMF Search** | Retention | 40%+ monthly |
| **Growth** | Revenue or users | 5-7% weekly |
| **Scale** | Unit economics | Positive CAC/LTV |

### Transition Criteria

```
Pre-launch → MVP:        Have something users can use
MVP → PMF Search:        Users are engaging
PMF Search → Growth:     Sean Ellis 40%+ ("very disappointed")
Growth → Scale:          Repeatable, profitable acquisition
```

## Key Metrics by Function

### User Metrics

| Metric | Formula | Good Target |
|--------|---------|-------------|
| **WAU/MAU** | Weekly Active / Monthly Active | 40%+ |
| **DAU/MAU** | Daily Active / Monthly Active | 20%+ |
| **Retention** | Users returning after 30 days | 40%+ |
| **Churn** | Users lost / Total users | <5%/month |
| **NPS** | Promoters - Detractors | 50+ |

### Revenue Metrics

| Metric | Formula | Good Target |
|--------|---------|-------------|
| **MRR Growth** | (MRR end - MRR start) / MRR start | 10%+ monthly |
| **LTV** | ARPU × Average lifetime | 3x+ CAC |
| **CAC** | Sales + Marketing / New customers | <1/3 LTV |
| **Payback Period** | CAC / Monthly Revenue | <12 months |
| **Gross Margin** | Revenue - COGS / Revenue | 60%+ SaaS |

### Engagement Metrics

| Metric | What It Measures | Good Signal |
|--------|------------------|-------------|
| **Time in app** | User engagement | Increasing |
| **Actions per session** | Value extraction | Increasing |
| **Feature adoption** | Product stickiness | 30%+ use key features |
| **Virality (K-factor)** | Organic growth | >1 = viral |

## Product-Market Fit Metrics

### The Sean Ellis Test

> "If you could no longer use [product], how would you feel?"

| Response | Threshold | Meaning |
|----------|-----------|---------|
| **Very disappointed** | 40%+ | You have PMF |
| **Somewhat disappointed** | N/A | Getting close |
| **Not disappointed** | N/A | No PMF yet |

### Other PMF Indicators

| Indicator | What to Look For |
|-----------|------------------|
| **Organic growth** | Users finding you without marketing |
| **Word of mouth** | Users actively recommending |
| **Pull > Push** | Customers seeking you out |
| **Retention curves** | Flatten, not decline to zero |
| **Revenue** | Customers paying, willingness to pay more |

### PMF Anti-Patterns

| Fake Signal | Why It's Misleading |
|-------------|---------------------|
| High signups | Doesn't mean retention |
| Press coverage | Doesn't mean product works |
| Investor interest | Investors aren't customers |
| Vanity metrics | Downloads ≠ engagement |

## Unit Economics

### The Core Equation

```
LTV > CAC

Specifically: LTV ≥ 3× CAC (for venture-scale)

Where:
- LTV = Average Revenue × Gross Margin × Customer Lifetime
- CAC = (Sales + Marketing) / New Customers Acquired
```

### Healthy Ratios

| Metric | Healthy Range |
|--------|---------------|
| LTV/CAC | 3:1 or higher |
| Payback Period | <12 months |
| Gross Margin | 60%+ (SaaS), 40%+ (e-commerce) |
| Net Revenue Retention | 100%+ (SaaS) |

### Common Issues

| Problem | Symptom | Solution |
|---------|---------|----------|
| **CAC too high** | Losing money on each customer | Improve conversion, find cheaper channels |
| **LTV too low** | Customers don't stick | Improve retention, add value |
| **Payback too long** | Cash flow problems | Raise prices, improve efficiency |

## Growth Frameworks

### The Startup Curve

```
Phase 1: Launch → Initial excitement
Phase 2: Trough of sorrow → Reality sets in
Phase 3: Wiggles of false hope → Temporary improvements
Phase 4: The long slow climb → Gradual, real progress
Phase 5: Growth → Sustainable acceleration
```

### Growth Loops (vs. Funnels)

**Traditional Funnel:**
```
Acquisition → Activation → Retention → Revenue → Referral
(Linear, requires constant input)
```

**Growth Loop:**
```
User → Action → Output → New User → (repeat)
(Self-reinforcing, compounds)
```

**Examples:**
- **Content loop:** User creates content → Content attracts new users
- **Referral loop:** User invites friend → Friend becomes user
- **Data loop:** User adds data → Product gets better → More users join

## Growth Myths vs. Reality

### What YC Has Learned

| Myth | Reality |
|------|---------|
| "Growth hacks scale" | Sustainable growth beats tricks |
| "We need more features" | Focus on core value first |

For social enterprises, track both:

| Business Metric | Impact Metric |
|-----------------|---------------|
| Revenue growth | Beneficiaries reached |
| Customer retention | Lives improved |

```
1. Prove impact first (Theory of Change validated)
2. Prove sustainability second (Unit economics work)

```

## Enterprise Transition Metrics

### When to Shift from Startup to Enterprise Mode

| Indicator | Startup Mode | Enterprise Mode |
|-----------|--------------|-----------------|
| PMF | Searching | Achieved (40%+) |
| Growth Rate | Variable | Consistent 5%+ weekly |
| Team Size | <15 | 15-50+ |
| Unit Economics | Finding | Positive |
| Customer Acquisition | Testing | Repeatable |

### Metrics Maturity

```
Early Stage:
- Track few metrics (users, revenue)
- Manual tracking acceptable
- Focus on direction, not precision

Growth Stage:
- Track more metrics (cohorts, segments)
- Automated dashboards required
- Focus on optimization

Scale Stage:
- Track comprehensive metrics
- Real-time analytics
- Focus on efficiency
```
