# Workshop Registration to Attendance Journey - MoroAcademy

## Overview
This journey maps the experience of a participant registering for a live workshop on MoroAcademy, attending the workshop sessions, accessing resources, and receiving their certificate. Live workshops are synchronous, scheduled training events that complement the self-paced online courses, often featuring hands-on activities and interactive facilitation.

## Persona
**Name**: Aisha Dimaporo
**Age**: 35
**Role**: Workshop Participant & Cooperative Secretary
**Context**: Aisha is the secretary of a women's cooperative in Maguindanao. Her cooperative was selected by CSEA to attend a 3-day "Financial Literacy Workshop" as part of a capacity-building program. She received a registration link from her cooperative's training officer and needs to register, attend all sessions, and obtain her certificate.
**Goals**:
- Register for the workshop before the deadline
- Attend all scheduled sessions
- Access workshop materials and templates
- Receive a valid completion certificate

## Journey Map

### Stage 1: Workshop Discovery & Registration Link
**User Goal**: Access the workshop registration page

**Actions**:
1. Receives workshop registration link from training officer or email
2. Clicks the link to access workshop page
3. Reviews workshop details (title, dates, facilitator)
4. Understands this is a scheduled, live workshop
5. Notes registration deadline and requirements

**Touchpoints**:
- Email or message with registration link
- Workshop landing page
- Workshop details section
- Schedule display
- Registration requirements

**Emotions**: Curious, slightly excited about learning opportunity
**Pain Points**:
- Link expired or broken
- Unclear if workshop is free or paid
- Not knowing how long sessions are
- Unclear prerequisites or requirements
- Confusing workshop vs course distinction

**Opportunities**:
- Clear workshop branding distinct from courses
- Prominent schedule with time zones
- Prerequisites clearly listed
- "Live Workshop" badge/indicator
- Registration deadline countdown
- Facilitator introduction

### Stage 2: Workshop Registration
**User Goal**: Complete registration and secure spot

**Actions**:
1. Clicks "Register Now" button
2. Logs in or creates MoroAcademy account
3. Fills registration form (organization, role)
4. Confirms schedule availability
5. Submits registration
6. Receives confirmation with details

**Touchpoints**:
- Registration form
- Account creation/login
- Registration confirmation page
- Confirmation email with calendar invite
- Access code delivery

**Emotions**: Committed, hopeful spot is available
**Pain Points**:
- Long registration form
- Not knowing if spot is confirmed
- No calendar integration
- Unclear what to do next
- Missing access code or instructions

**Opportunities**:
- Minimal registration fields
- Real-time spot availability display
- Calendar invite attachment (.ics)
- Clear next steps instructions
- Access code sent immediately
- SMS reminder option

### Stage 3: Pre-Workshop Preparation
**User Goal**: Prepare for workshop with required materials

**Actions**:
1. Receives reminder emails before workshop
2. Reviews pre-workshop materials (if any)
3. Downloads preparation documents
4. Notes workshop schedule and platform
5. Tests technology (if online component)
6. Saves access code for attendance

**Touchpoints**:
- Reminder emails (1 week, 1 day before)
- Pre-workshop resources page
- Downloadable materials
- Technology requirements
- Access code reminder

**Emotions**: Anticipating, preparing, slightly nervous
**Pain Points**:
- No reminders sent
- Pre-materials sent too late
- Unclear technology requirements
- Lost access code
- No way to ask pre-workshop questions

**Opportunities**:
- Automated reminder sequence
- Pre-materials available immediately after registration
- Clear tech requirements and test page
- Access code prominently displayed in dashboard
- Pre-workshop FAQ or chat
- WhatsApp/Viber group for participants

### Stage 4: Workshop Access & Attendance
**User Goal**: Join workshop sessions and mark attendance

**Actions**:
1. On workshop day, accesses MoroAcademy
2. Navigates to "My Workshops" or dashboard
3. Enters access code to mark attendance
4. Joins live session (if virtual) or notes venue (if in-person)
5. Attendance is recorded for the day
6. Repeats for each workshop day

**Touchpoints**:
- Workshop dashboard/My Workshops
- Access code entry interface
- Attendance confirmation
- Live session link (if virtual)
- Venue details (if in-person)

**Emotions**: Engaged, focused on learning
**Pain Points**:
- Can't find where to enter code
- Access code not working
- No confirmation of attendance
- Missing live session link
- Unclear if all sessions are required

**Opportunities**:
- Prominent "Enter Code" button on workshop day
- Real-time attendance confirmation
- Attendance tracker (Day 1, Day 2, Day 3)
- Integrated live session join button
- Clear session schedule within workshop
- "You're registered" status display

### Stage 5: Workshop Participation
**User Goal**: Actively participate in workshop activities

**Actions**:
1. Follows facilitator's presentation
2. Participates in group activities
3. Uses provided templates or worksheets
4. Takes notes on key learnings
5. Asks questions during Q&A
6. Completes in-session exercises

**Touchpoints**:
- Live session (Zoom, Google Meet, or in-person)
- Workshop materials page
- Downloadable templates
- Interactive tools (if virtual)
- Q&A or chat function

**Emotions**: Engaged, collaborative, learning actively
**Pain Points**:
- Materials not accessible during session
- Templates hard to find
- Can't download materials offline
- No way to save notes
- Audio/video issues (virtual)

**Opportunities**:
- Materials available before and during session
- Mobile-friendly material access
- Offline download option
- Integrated note-taking
- Virtual workshop tools (polls, breakout rooms)
- Recording availability for review

### Stage 6: Session Materials & Resources
**User Goal**: Access and use workshop templates and resources

**Actions**:
1. Accesses workshop resources section
2. Downloads workshop slides (if provided)
3. Downloads templates (budget templates, forms)
4. Reviews supplementary reading materials
5. Saves resources for post-workshop use
6. Shares relevant materials with cooperative

**Touchpoints**:
- Resources tab within workshop
- Download buttons for each resource
- Resource descriptions
- File format indicators
- Share functionality

**Emotions**: Resourceful, planning to apply learning
**Pain Points**:
- Resources scattered or hard to find
- No clear labeling of files
- Can't preview before downloading
- Resources removed after workshop
- No mobile access to resources

**Opportunities**:
- Organized resources by session/topic
- Clear file names and descriptions
- Preview capability
- Permanent resource access (not just during workshop)
- Mobile-optimized downloads
- "Share with Team" option

### Stage 7: Workshop Completion
**User Goal**: Complete all workshop requirements

**Actions**:
1. Attends all required sessions
2. Completes any required exercises or submissions
3. Views completion status in dashboard
4. Confirms all attendance recorded
5. Checks eligibility for certificate
6. Provides workshop feedback (evaluation)

**Touchpoints**:
- Attendance tracker with all sessions
- Exercise submission interface (if any)
- Completion status indicator
- Certificate eligibility display
- Feedback/evaluation form

**Emotions**: Accomplished, ready for certification
**Pain Points**:
- Missing attendance not flagged
- Unclear what "complete" means
- No feedback collection
- Uncertain if certificate will be issued
- Can't check completion status

**Opportunities**:
- Clear attendance and completion tracker
- Early warning if missing requirements
- Quick feedback form (not too long)
- Automatic completion verification
- "You're eligible for certificate" confirmation
- Thank you message from facilitator

### Stage 8: Certificate Issuance
**User Goal**: Receive official workshop completion certificate

**Actions**:
1. Receives notification that certificate is ready
2. Accesses certificate in My Certificates section
3. Reviews certificate details (name, workshop, date)
4. Downloads certificate PDF
5. Notes verification code/QR
6. Shares certificate with cooperative and CSEA

**Touchpoints**:
- Certificate ready notification
- My Certificates page
- Certificate preview
- Download button (PDF)
- Verification code/QR display
- Share options

**Emotions**: Proud, validated, ready to showcase
**Pain Points**:
- Certificate not automatically generated
- Long wait for certificate
- Name spelled incorrectly
- Certificate looks unofficial
- Can't verify authenticity

**Opportunities**:
- Immediate certificate generation after completion
- Name verification before issuance
- Professional, branded certificate design
- QR code for verification
- Social sharing options
- Direct email to cooperative option

### Stage 9: Post-Workshop Follow-Up
**User Goal**: Apply learning and maintain connection

**Actions**:
1. Receives post-workshop follow-up email
2. Accesses workshop recording (if applicable)
3. Reviews additional resources shared
4. Joins alumni network or community (if any)
5. Receives information about related courses
6. Applies workshop learnings in cooperative

**Touchpoints**:
- Follow-up email sequence
- Recording access (if virtual)
- Post-workshop resources
- Community/network invitation
- Related course recommendations

**Emotions**: Empowered, connected, implementing
**Pain Points**:
- No follow-up after workshop
- Recording not available
- No community for continued learning
- No guidance on next steps
- Forgotten after certificate

**Opportunities**:
- Automated follow-up sequence
- Recording access for 30-60 days
- Alumni network or group
- Suggested next courses
- Implementation check-in survey
- Advanced workshop notifications

### Phase 3: Backend Integration

- `GET /api/training/workshops` - List workshops (upcoming, past, by category)
- `GET /api/training/workshops/{slug}` - Workshop detail with sessions
- `POST /api/training/workshops/{slug}/register` - Register for workshop
- `GET /api/training/workshops/my` - User's registered workshops
- `GET /api/training/workshops/{id}/sessions` - Workshop sessions detail
- `POST /api/training/workshops/{id}/attendance` - Submit access code
- `GET /api/training/workshops/{id}/attendance` - Get attendance status
- `GET /api/training/workshops/{id}/resources` - Workshop materials
- `POST /api/training/workshops/{id}/feedback` - Submit feedback
- `GET /api/training/workshops/{id}/certificate` - Get workshop certificate
- `POST /api/training/workshops/{id}/certificate/generate` - Generate certificate

- `RegistrationCreateSchema` - Registration request payload

- `AttendanceSubmitSchema` - Access code submission
- `AttendanceStatusSchema` - Attendance by session
- `WorkshopResourceSchema` - Resource file details
- `WorkshopFeedbackSchema` - Evaluation submission
- `WorkshopCertificateSchema` - Certificate with verification

**Models** (in `backend/apps/training/models.py`):
