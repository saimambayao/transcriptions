# Instructor Course Creation Journey - MoroAcademy

## Overview
This journey maps the experience of a training provider or instructor creating and publishing a course on MoroAcademy. It covers course planning, content creation, assessment setup, publishing, and monitoring learner engagement. The focus is on making course creation intuitive while maintaining quality standards.

## Persona
**Name**: Dr. Hassan Lucman
**Age**: 45
**Role**: Training Consultant & Accredited TSP
**Context**: Dr. Hassan is an independent training service provider (TSP) accredited by CSEA. He has 15 years of experience in cooperative development and has been conducting in-person workshops. He wants to expand his reach by creating online courses on MoroAcademy, starting with his signature "Strategic Planning for Cooperatives" workshop.
**Goals**:
- Convert his in-person workshop into an online course
- Reach more cooperatives across Bangsamoro
- Generate passive income from course sales
- Establish his reputation as a leading trainer

## Journey Map

### Stage 1: Provider Onboarding & Access
**User Goal**: Get approved as a course creator on MoroAcademy

**Actions**:
1. Logs into MoroAcademy with TSP account
2. Navigates to "Instructor Dashboard" or "Create Course"
3. If not yet approved, completes provider verification
4. Submits credentials and accreditation documents
5. Receives approval notification
6. Accesses course creation tools

**Touchpoints**:
- MoroAcademy login
- Instructor dashboard (/academy/instructor)
- Provider verification form
- Approval notification
- Course creation CTA

**Emotions**: Eager to start, slightly uncertain about process
**Pain Points**:
- Unclear how to become a course creator
- Long approval wait time
- Not knowing approval criteria
- No guidance on what makes a good course

**Opportunities**:
- Clear "Become an Instructor" pathway
- Transparent approval criteria
- Fast-track for CSEA-accredited TSPs
- Onboarding tutorial for new instructors
- Sample courses to reference

### Stage 2: Course Planning & Setup
**User Goal**: Set up course structure and basic information

**Actions**:
1. Clicks "Create New Course" button
2. Enters course title and subtitle
3. Writes course description and learning objectives
4. Selects course category (Cooperative Development)
5. Sets course level (Beginner, Intermediate, Advanced)
6. Defines target audience
7. Sets pricing (free or paid)
8. Uploads course thumbnail image
9. Saves draft course

**Touchpoints**:
- Create course wizard/form
- Course details page (title, description)
- Category selector
- Pricing configuration
- Image upload
- Draft save confirmation

**Emotions**: Excited about creating, focused on positioning
**Pain Points**:
- Blank page intimidation
- Unsure what makes compelling description
- Pricing uncertainty
- Image size/format requirements unclear
- No template or guidance

**Opportunities**:
- Course creation wizard with prompts
- AI-assisted description suggestions
- Pricing recommendations based on market
- Image templates and guidelines
- "Save as Draft" auto-save feature
- Preview of how course will appear

### Stage 3: Curriculum Structure
**User Goal**: Organize course into logical modules and lessons

**Actions**:
1. Accesses curriculum builder
2. Creates modules (sections) for course
3. Adds lessons within each module
4. Reorders modules and lessons via drag-drop
5. Marks which lessons are "free preview"
6. Estimates time for each lesson
7. Saves curriculum structure

**Touchpoints**:
- Curriculum builder page
- Module creation interface
- Lesson creation interface
- Drag-and-drop reordering
- Preview checkbox
- Time estimation input

**Emotions**: Organized, slightly overwhelmed by structure decisions
**Pain Points**:
- Blank curriculum intimidating
- Unclear optimal course structure
- Can't visualize final navigation
- No template to follow
- Difficult to reorder items

**Opportunities**:
- Template curricula to start from
- "Recommended structure" suggestions
- Live preview of learner navigation
- Easy drag-and-drop reordering
- Progress indicator for curriculum completion
- AI suggestions for module organization

### Stage 4: Content Creation - Video Lessons
**User Goal**: Upload and configure video lessons

**Actions**:
1. Selects lesson to edit
2. Chooses "Video" as lesson type
3. Uploads video file (or provides URL)
4. Waits for video processing
5. Adds video description and supplementary notes
6. Attaches downloadable resources
7. Sets lesson as required or optional
8. Previews lesson
9. Saves and moves to next lesson

**Touchpoints**:
- Lesson editor page
- Video upload interface
- Upload progress indicator
- Processing status
- Description text editor
- Resource attachment
- Preview button

**Emotions**: Patient during upload, proud of content
**Pain Points**:
- Large file upload failures
- Long processing times
- Unclear video requirements (format, size, quality)
- Can't preview before full processing
- No video editing tools
- Losing progress on connection issues

**Opportunities**:
- Chunked upload for large files
- Clear video requirements upfront
- Processing progress indicator
- Preview during processing
- Resume interrupted uploads
- Video quality recommendations
- Subtitle/caption support

### Stage 5: Content Creation - Text & Resources
**User Goal**: Create text lessons and attach resources

**Actions**:
1. Selects lesson to edit
2. Chooses "Text" as lesson type
3. Uses rich text editor to create content
4. Adds images, diagrams, or embedded media
5. Attaches downloadable files (PDFs, templates)
6. Formats content for readability
7. Previews lesson
8. Saves lesson

**Touchpoints**:
- Text lesson editor
- Rich text toolbar
- Image upload/embed
- File attachment section
- Preview mode
- Save button

**Emotions**: Creative, focused on clarity
**Pain Points**:
- Limited formatting options
- Image sizing issues
- Can't embed certain media
- Preview doesn't match final
- Formatting lost on paste

**Opportunities**:
- Full-featured rich text editor
- Easy image resize and alignment
- Embed support (YouTube, SlideShare)
- WYSIWYG preview accuracy
- Clean paste from Word/Google Docs
- Template blocks for common patterns

### Stage 6: Assessment Creation
**User Goal**: Create quizzes to assess learner understanding

**Actions**:
1. Adds quiz to module
2. Sets quiz title and instructions
3. Creates questions (multiple choice, true/false)
4. Adds answer options with correct answer marked
5. Writes feedback for each answer (optional)
6. Sets passing score threshold
7. Configures attempts allowed
8. Previews quiz
9. Saves quiz

**Touchpoints**:
- Quiz builder interface
- Question creation form
- Answer options with correct marker
- Feedback text fields
- Quiz settings (pass score, attempts)
- Preview mode

**Emotions**: Focused on fairness and effectiveness
**Pain Points**:
- Limited question types
- Tedious to create many questions
- Can't bulk import questions
- No question bank to reuse
- Unclear what makes good quiz

**Opportunities**:
- Multiple question types (matching, fill-blank)
- Bulk question import (CSV, Excel)
- Question bank for reuse
- AI-generated question suggestions
- Quiz best practices guide
- Question randomization option

### Stage 7: Course Review & Polish
**User Goal**: Review entire course before publishing

**Actions**:
1. Reviews course completeness checklist
2. Previews course as a learner would see it
3. Tests video playback and quiz functionality
4. Reviews curriculum flow and timing
5. Checks all resources are attached
6. Reviews course description and thumbnail
7. Makes final edits

**Touchpoints**:
- Completeness checklist dashboard
- Learner preview mode
- Course overview summary
- Edit links for each section

**Emotions**: Meticulous, slightly nervous about quality
**Pain Points**:
- No clear checklist of requirements
- Preview mode not accurate
- Can't easily identify missing items
- No spell-check or grammar review
- Uncertain about quality standards

**Opportunities**:
- Automated completeness checker
- Accurate learner preview
- Clear quality guidelines
- Content warnings (empty lessons, short modules)
- Peer review option (optional)
- Quality score indicator

### Stage 8: Course Publishing
**User Goal**: Publish course and make it available to learners

**Actions**:
1. Clicks "Submit for Review" or "Publish"
2. Reviews publishing requirements
3. Confirms course is ready
4. Submits course (may require review for first courses)
5. Receives confirmation of submission/publication
6. Views course on public catalog
7. Shares course link for promotion

**Touchpoints**:
- Publish button with confirmation
- Requirements checklist
- Submission confirmation
- Publication status indicator
- Public course page
- Share tools

**Emotions**: Excited, proud, eager for learners
**Pain Points**:
- Unexpected review delay
- Unclear review criteria
- Rejection without clear feedback
- Can't edit after publishing
- No promotion guidance

**Opportunities**:
- Clear publishing requirements upfront
- Fast review for verified TSPs
- Specific feedback if changes needed
- Easy post-publish edits
- Promotion tips and tools
- Social share buttons

### Stage 9: Monitoring & Improvement
**User Goal**: Track learner engagement and improve course

**Actions**:
1. Accesses instructor analytics dashboard
2. Views enrollment numbers over time
3. Checks completion rates per module
4. Identifies drop-off points
5. Reads learner reviews and ratings
6. Makes course improvements based on data
7. Responds to learner questions

**Touchpoints**:
- Instructor analytics dashboard
- Enrollment chart
- Completion funnel
- Review/rating display
- Course edit interface
- Q&A or message center

**Emotions**: Curious about performance, motivated to improve
**Pain Points**:
- Limited analytics available
- No insight into why learners drop off
- Can't respond to reviews
- No Q&A feature
- Unclear what improvements would help

**Opportunities**:
- Rich analytics dashboard
- Video engagement heatmaps
- Drop-off analysis with suggestions
- Review response capability
- Q&A forum for courses
- A/B testing for improvements

## Success Metrics
- **Course Creation Completion Rate**: % of started courses that get published
- **Time to First Publish**: Days from account creation to first course
- **Average Course Quality Score**: Based on completeness checklist
- **Instructor Satisfaction**: Survey scores from creators
- **Course Performance**: Enrollment and completion rates
- **Instructor Retention**: % who create additional courses

### Phase 3: Backend Integration
**Django Ninja Endpoints** (in `backend/apps/training/api.py`):
- `GET /api/training/instructor/courses` - List instructor's courses
- `POST /api/training/instructor/courses` - Create new course draft
- `GET /api/training/instructor/courses/{id}` - Get course for editing
- `PATCH /api/training/instructor/courses/{id}` - Update course details
- `DELETE /api/training/instructor/courses/{id}` - Delete draft course
- `POST /api/training/instructor/courses/{id}/modules` - Add module
- `PATCH /api/training/instructor/courses/{id}/modules/{moduleId}` - Update module
- `POST /api/training/instructor/courses/{id}/lessons` - Add lesson
- `PATCH /api/training/instructor/lessons/{lessonId}` - Update lesson content
- `POST /api/training/instructor/lessons/{lessonId}/upload` - Upload video
- `POST /api/training/instructor/quizzes` - Create quiz
- `PATCH /api/training/instructor/quizzes/{quizId}` - Update quiz questions
- `POST /api/training/instructor/courses/{id}/publish` - Submit for review/publish
- `GET /api/training/instructor/courses/{id}/analytics` - Get course analytics
- `POST /api/training/instructor/reviews/{reviewId}/respond` - Respond to review
