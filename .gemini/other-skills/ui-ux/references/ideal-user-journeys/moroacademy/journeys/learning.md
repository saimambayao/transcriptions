# Course Learning to Completion Journey - MoroAcademy

## Overview
This journey maps the experience of an enrolled learner progressing through a course on MoroAcademy, from starting their first lesson to completing the course and receiving their certificate. The focus is on maintaining engagement, tracking progress, and celebrating achievement.

## Persona
**Name**: Ahmad Pangandaman
**Age**: 32
**Role**: Enrolled Learner & Cooperative Treasurer
**Context**: Ahmad is the treasurer of an agricultural cooperative in Lanao del Sur. He enrolled in the "Financial Management for Cooperatives" course two days ago to improve his bookkeeping and financial reporting skills. He has limited time for learning, usually studying in the evenings after work.
**Goals**:
- Complete the course within 4 weeks
- Understand cooperative financial statements
- Pass the final assessment with a high score
- Earn a certificate to present to his cooperative board

## Journey Map

### Stage 1: Course Access & Orientation
**User Goal**: Access enrolled course and understand how to navigate it

**Actions**:
1. Logs into MoroAcademy account
2. Navigates to "My Courses" or learning dashboard
3. Clicks on enrolled course to enter
4. Views course welcome/orientation section
5. Understands course structure, timeline, and requirements
6. Locates first lesson and starts learning

**Touchpoints**:
- Login page
- Learning dashboard (/academy/learn)
- My Courses list
- Course player page
- Course navigation sidebar
- Welcome module

**Emotions**: Eager to start, slightly anxious about time commitment
**Pain Points**:
- Forgetting where to find enrolled courses
- Unclear course navigation structure
- No sense of expected time investment
- Overwhelming syllabus

**Opportunities**:
- "Continue Learning" prominent on dashboard
- Quick course overview card showing progress
- Estimated time per module/lesson
- Simplified navigation with clear hierarchy
- "Start Here" indicator for first lesson

### Stage 2: Content Consumption - Video Lessons
**User Goal**: Watch video lessons and absorb key concepts

**Actions**:
1. Clicks on video lesson
2. Watches video with playback controls
3. Adjusts playback speed if needed
4. Takes notes (mental or written)
5. Pauses to reflect on key points
6. Marks lesson as complete
7. Proceeds to next lesson

**Touchpoints**:
- Video player component
- Playback controls (speed, CC, fullscreen)
- Progress bar within video
- "Mark Complete" button
- "Next Lesson" navigation
- Lesson notes section (optional)

**Emotions**: Focused, engaged, sometimes distracted
**Pain Points**:
- Video buffering or quality issues
- No way to take notes within platform
- Losing place if interrupted
- No way to bookmark specific moments
- Audio-only option missing for commute learning

**Opportunities**:
- Adaptive video quality based on connection
- Integrated note-taking alongside video
- Auto-save progress within video (timestamp)
- Video bookmarks/highlights feature
- Audio download for offline listening
- Transcripts for accessibility

### Stage 3: Content Consumption - Text/Reading Lessons
**User Goal**: Read and understand text-based course materials

**Actions**:
1. Opens text lesson
2. Reads through content
3. Views embedded images, diagrams, or examples
4. Downloads supplementary materials (PDFs, templates)
5. Highlights or bookmarks important sections
6. Marks lesson as complete

**Touchpoints**:
- Text lesson page
- Reading progress indicator
- Download buttons for attachments
- Highlight/bookmark tools
- Completion checkbox

**Emotions**: Studious, sometimes fatigued from reading
**Pain Points**:
- Long blocks of text without visual breaks
- No way to highlight or annotate
- Missing context for downloaded files
- Unclear when reading is "complete"

**Opportunities**:
- Clean, readable typography with visual breaks
- Progress indicator for long lessons
- Inline highlighting and annotation tools
- Clear file descriptions and previews
- Estimated reading time display
- "Key Takeaways" summary boxes

### Stage 4: Knowledge Check - Quizzes
**User Goal**: Test understanding and reinforce learning through quizzes

**Actions**:
1. Reaches quiz section within module
2. Reads quiz instructions and requirements
3. Answers multiple choice/true-false questions
4. Submits quiz for grading
5. Reviews results and feedback
6. Retakes quiz if needed (if allowed)
7. Proceeds after passing

**Touchpoints**:
- Quiz introduction page
- Question display (one at a time or all)
- Answer selection interface
- Submit button with confirmation
- Results page with score
- Feedback per question (correct/incorrect)

**Emotions**: Slightly nervous, curious about performance, relieved after passing
**Pain Points**:
- Unclear how many attempts allowed
- No feedback on wrong answers
- Questions unclear or ambiguous
- No way to review after completion
- Passing threshold unclear

**Opportunities**:
- Clear quiz rules upfront (attempts, passing score)
- Immediate feedback with explanations
- Question review mode after submission
- Progress toward passing clearly shown
- Encouragement messages for retakes
- Link to relevant lesson for wrong answers

### Stage 5: Progress Tracking
**User Goal**: Monitor overall course progress and stay motivated

**Actions**:
1. Views progress dashboard/overview
2. Checks completion percentage
3. Reviews which modules are complete vs. remaining
4. Sees estimated time to completion
5. Sets personal goals or reminders
6. Celebrates milestones (module completion)

**Touchpoints**:
- Course progress bar
- Module completion indicators
- Overall completion percentage
- Time tracking (if enabled)
- Milestone notifications
- Learning streak display

**Emotions**: Motivated by progress, sometimes discouraged if behind
**Pain Points**:
- Progress not updating correctly
- No sense of achievement for milestones
- Unclear what counts toward completion
- No reminders to continue learning
- Can't see how far behind schedule

**Opportunities**:
- Real-time progress updates
- Module completion celebrations (confetti, badges)
- Weekly progress email summaries
- Smart reminders based on learning pattern
- Catch-up suggestions if falling behind
- Comparison to average learner pace

### Stage 6: Module Completion
**User Goal**: Complete individual modules and feel sense of accomplishment

**Actions**:
1. Finishes all lessons in a module
2. Completes module quiz or assessment
3. Views module completion confirmation
4. Unlocks next module (if gated)
5. Reviews module summary or key takeaways
6. Proceeds to next module

**Touchpoints**:
- Module completion screen
- "Next Module" navigation
- Module certificate/badge (optional)
- Key takeaways summary
- Unlock notification for next module

**Emotions**: Accomplished, motivated to continue
**Pain Points**:
- No recognition of module completion
- Abrupt transition to next module
- No summary of what was learned
- Unclear if module assessment is required

**Opportunities**:
- Module completion celebration
- Summary of key concepts mastered
- "Share Progress" social option
- Badge or micro-credential per module
- Preview of next module to maintain interest

### Stage 7: Course Completion
**User Goal**: Complete entire course and receive certificate

**Actions**:
1. Finishes final module and assessment
2. Views course completion confirmation
3. Reviews final score or grade
4. Generates/downloads course certificate
5. Shares achievement (if desired)
6. Provides course feedback/review
7. Explores related courses

**Touchpoints**:
- Course completion page
- Final score summary
- Certificate generation button
- Download certificate (PDF)
- Share buttons (LinkedIn, Facebook)
- Course review/rating form
- Recommended next courses

**Emotions**: Proud, accomplished, eager to showcase achievement
**Pain Points**:
- Certificate not immediately available
- Certificate looks unprofessional
- No way to verify certificate authenticity
- No guidance on what to do next
- Forced to leave review

**Opportunities**:
- Immediate certificate generation
- Professional, branded certificate design
- Unique certificate ID for verification
- LinkedIn integration for sharing
- Suggested learning paths
- Optional but encouraged review
- "What I Learned" summary page

## Success Metrics
- **Course Completion Rate**: % of enrollees who complete the course
- **Lesson Completion Rate**: % of lessons marked complete
- **Quiz Pass Rate**: % of quizzes passed on first attempt
- **Average Time to Completion**: Days from enrollment to completion
- **Learner Satisfaction**: Post-course survey scores
- **Engagement Rate**: % of active learners per week
- **Drop-off Points**: Where learners abandon the course

## Recommendations

### Priority 1: Core Learning Experience
1. **Robust Video Player**: Implement reliable video playback with speed controls, quality options, and progress saving
2. **Clear Progress Indicators**: Show completion percentage, module status, and remaining content clearly
3. **Quiz Feedback System**: Provide immediate, helpful feedback on quiz answers with explanations

### Priority 2: Engagement & Retention
4. **Lesson Completion Tracking**: Auto-track video completion, require explicit mark-complete for text
5. **Progress Celebrations**: Add micro-celebrations for milestones (module complete, quiz passed)
6. **Learning Reminders**: Email or push notifications for inactive learners

### Priority 3: Completion & Certification
7. **Certificate Generation**: Create professional, verifiable certificates on course completion
8. **Course Summary Page**: Show what was learned and skills gained upon completion
9. **Social Sharing**: Enable easy sharing of achievements to LinkedIn and social media

### Priority 4: Enhanced Features
10. **Note-Taking Integration**: Allow learners to take notes alongside content
11. **Bookmarks**: Let learners save specific lessons or video timestamps
12. **Audio Downloads**: Enable audio-only download for offline learning
13. **Learning Streaks**: Gamify consistent learning with streak tracking

- `video-player.tsx` - Video player with controls (speed, quality, fullscreen)
- `text-lesson.tsx` - Rich text content display with estimated read time
- `quiz-player.tsx` - Quiz question display and submission
- `quiz-results.tsx` - Quiz score and answer review
- `progress-bar.tsx` - Overall course progress indicator
- `lesson-complete-button.tsx` - Mark lesson as complete CTA
- `module-completion.tsx` - Module completion celebration overlay
- `course-completion.tsx` - Final completion screen with certificate CTA
- `continue-learning-card.tsx` - Resume course card for dashboard
- `learning-streak.tsx` - Streak counter and celebration

### Phase 2: State & Services
**Hooks** (under `frontend/src/lib/hooks/academy/...`):
- `use-enrolled-courses.ts` - TanStack Query hook for user's enrolled courses
- `use-course-progress.ts` - Hook for tracking and updating lesson progress
- `use-lesson-content.ts` - Hook for fetching lesson content (video/text)
- `use-quiz.ts` - Hook for quiz submission and results
- `use-certificate.ts` - Hook for certificate generation status

**Services** (under `frontend/src/lib/services/academy/...`):
- `progress-service.ts` - API calls for progress tracking
- `lesson-service.ts` - API calls for lesson content
- `quiz-service.ts` - API calls for quiz submission
- `certificate-service.ts` - API calls for certificate generation

**Types** (under `frontend/src/lib/types/academy/...`):
- `progress.ts` - LessonProgress, ModuleProgress, CourseProgress types
- `quiz.ts` - Question, Answer, QuizAttempt, QuizResult types
- `certificate.ts` - Certificate data and verification types

### Phase 3: Backend Integration
**Django Ninja Endpoints** (in `backend/apps/training/api.py`):
- `GET /api/training/enrollments/me` - Get user's enrolled courses with progress
- `GET /api/training/courses/{slug}/lessons/{lessonSlug}` - Get lesson content
- `POST /api/training/progress/{lessonId}/complete` - Mark lesson complete
- `GET /api/training/courses/{slug}/progress` - Get detailed course progress
- `POST /api/training/quizzes/{quizId}/submit` - Submit quiz answers
- `GET /api/training/quizzes/{quizId}/results` - Get quiz results with feedback
- `POST /api/training/courses/{slug}/certificate` - Generate certificate
- `GET /api/training/certificates/{id}` - Get certificate details

**Schemas** (in `backend/apps/training/schemas.py`):
- `EnrolledCourseSchema` - Enrolled course with progress summary
- `LessonContentSchema` - Full lesson content (video URL or text)
- `ProgressUpdateSchema` - Progress update payload
- `QuizSubmissionSchema` - Quiz answer submission
- `QuizResultSchema` - Quiz results with correct answers
- `CertificateSchema` - Certificate details for display/download

**Models Enhancement** (in `backend/apps/training/models.py`):
- `LessonProgress` - Track individual lesson completion
- `QuizAttempt` - Store quiz submissions and scores
- `Certificate` - Store issued certificates with verification ID
