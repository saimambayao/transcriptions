## Assessment Fundamentals

### Purpose of Assessment

**Formative Assessment** (During Learning):
- Check understanding
- Identify gaps
- Provide feedback
- Guide instruction
- Examples: Lesson quizzes, discussion questions, polls

**Summative Assessment** (End of Learning):
- Measure achievement
- Certify competency
- Grade performance
- Examples: Final exams, projects, portfolios

### Alignment Principle

Every assessment item must connect to a learning objective:

```
Objective → Content → Assessment

"Calculate patronage refund" → Calculation lesson → Quiz: Calculate refund given scenario
"Explain cooperative principles" → Principles video → Quiz: Describe principle X
"Design business plan" → Planning module → Project: Submit business plan
```

### Validity and Reliability

**Validity**: Does the assessment measure what it should?
- Content validity: Covers the material taught
- Face validity: Looks like a fair test
- Criterion validity: Predicts real-world performance

**Reliability**: Does the assessment give consistent results?
- Clear instructions
- Unambiguous questions
- Objective scoring criteria
- Sufficient number of items

## Quiz Question Patterns

### Multiple Choice Questions

**Structure**:
```
Stem: Clear question or incomplete statement
Options: One correct answer + 3-4 plausible distractors
```

**Good Example**:
```
Which cooperative principle emphasizes that cooperatives are
autonomous organizations controlled by their members?

A. Open and Voluntary Membership
B. Democratic Member Control ← Correct
C. Education, Training, and Information
D. Concern for Community
```

**Writing Guidelines**:
- Stem should make sense without reading options
- All options similar in length and complexity
- Avoid "all of the above" and "none of the above"
- Avoid negative phrasing
- Random position for correct answer

### True/False Questions

**Good Example**:
```
A cooperative can distribute surplus to non-members.

○ True
○ False ← Correct

Feedback: Surplus distribution is limited to members
based on their patronage, per cooperative principles.
```

**Writing Guidelines**:
- One clear statement per question
- Avoid double negatives
- Balance true and false answers (50/50)
- Include explanation in feedback

### Fill-in-the-Blank

**Good Example**:
```
The formula for calculating patronage refund is:
Total Surplus × (Member's Patronage ÷ ________)

Answer: Total Patronage
```

**Writing Guidelines**:
- Only one blank per question
- Blank at end when possible
- Clear what type of answer expected
- Accept reasonable variations

### Matching Questions

**Good Example**:
```
Match each cooperative officer with their primary responsibility:

Officers:                  Responsibilities:
1. Chairperson      A. Financial records and reports
2. Treasurer        B. Meeting facilitation
3. Secretary        C. Minutes and documentation
4. Manager          D. Daily operations

Answers: 1-B, 2-A, 3-C, 4-D
```

**Writing Guidelines**:
- Homogeneous items (all same category)
- More options than matches (prevents guessing last one)
- Clear matching relationship
- 5-8 items maximum

### Short Answer Questions

**Good Example**:
```
In 2-3 sentences, explain why cooperatives maintain
a reserve fund.

Sample Answer: Reserve funds provide financial stability
by covering unexpected losses or emergencies. They also
demonstrate the cooperative's financial health to lenders
and regulators, building credibility.

Scoring: 2 points for financial stability concept,
2 points for credibility/compliance concept.
```

**Writing Guidelines**:
- Clear word/sentence limit
- Specific focus (not too broad)
- Sample answer for scorer
- Clear scoring criteria

### Scenario-Based Questions

**Good Example**:
```
Scenario: Maguindanao Agricultural Cooperative had a total
surplus of P500,000 this year. Member Juan purchased P50,000
worth of products, while total member patronage was P2,500,000.
The board allocated 70% of surplus for patronage refund.

Question: Calculate Juan's patronage refund.

A. P5,000
B. P7,000 ← Correct
C. P10,000
D. P35,000

Solution:
Refund = (500,000 × 0.70) × (50,000 ÷ 2,500,000)
       = 350,000 × 0.02 = P7,000
```

**Writing Guidelines**:
- Realistic, relevant scenario
- Clear, specific data
- One correct calculation path
- Distractors based on common errors

## Practical Assessments

### Project-Based Assessment

**Design Template**:
```markdown
## Project: [Title]

**Objective**: Demonstrate ability to [specific skill]

**Description**: [What participants will create]

**Requirements**:
- [ ] Requirement 1
- [ ] Requirement 2
- [ ] Requirement 3

**Submission Format**: [Document type, file format]

**Due Date**: [When]

**Evaluation Criteria**: See rubric below
```

**Example - Business Plan Project**:
```markdown
## Project: Cooperative Business Plan

**Objective**: Develop a viable business plan for a
cooperative venture

**Requirements**:
- [ ] Executive summary (1 page)
- [ ] Market analysis with local data
- [ ] Product/service description
- [ ] Financial projections (3 years)
- [ ] Cooperative governance structure
- [ ] Risk analysis and mitigation

**Submission**: PDF document, max 15 pages
```

### Performance Assessment

**Observation Checklist**:
```
Task: Conduct a General Assembly

Observer: ___________  Date: ___________

□ Called meeting to order on time
□ Verified quorum present
□ Followed agenda order
□ Allowed member participation
□ Managed time appropriately
□ Conducted voting properly
□ Summarized decisions
□ Adjourned formally

Score: ___/8 items completed correctly
```

### Portfolio Assessment

**Portfolio Components**:
```
Required Items:
1. Reflection essay on learning journey
2. Three completed worksheets/exercises
3. One case study analysis
4. Final project output
5. Self-assessment checklist

Optional Items:
- Additional practice exercises
- Notes from group discussions
- Photos/documentation from activities
```

## Rubric Design

### Analytic Rubric

Scores each criterion separately:

```
Business Plan Rubric (20 points total)

MARKET ANALYSIS (5 points)
5 - Comprehensive local data, clear target market,
    competitor analysis included
4 - Good local data, defined target market,
    some competitor info
3 - Basic market information, general target market
2 - Limited market data, vague target
1 - No market analysis or irrelevant data

FINANCIAL PROJECTIONS (5 points)
5 - Realistic 3-year projections, clear assumptions,
    sensitivity analysis
4 - Reasonable projections, stated assumptions
3 - Basic projections, some assumptions unclear
2 - Unrealistic or incomplete projections
1 - No financial projections

GOVERNANCE STRUCTURE (5 points)
5 - Clear org chart, roles defined, aligned with
    cooperative principles
4 - Good structure, most roles clear
3 - Basic structure, some roles undefined
2 - Incomplete structure
1 - No governance structure

PRESENTATION (5 points)
5 - Professional format, clear writing, well-organized
4 - Good format, clear writing
3 - Acceptable format, some clarity issues
2 - Poor format or unclear writing
1 - Unprofessional presentation
```

### Holistic Rubric

Single overall score:

```
Workshop Output Rubric

EXCELLENT (4)
Output fully addresses the task requirements with
depth and insight. Shows creative application of
concepts. Presentation is professional and clear.
No significant gaps or errors.

PROFICIENT (3)
Output addresses all major requirements. Shows
solid understanding of concepts. Presentation
is clear. Minor gaps or errors present.

DEVELOPING (2)
Output addresses some requirements but missing
key elements. Shows basic understanding. Some
presentation issues. Notable gaps or errors.

BEGINNING (1)
Output is incomplete or does not address the
task. Limited understanding demonstrated.
Significant presentation issues or errors.
```

### Checklist Rubric

Binary criteria (yes/no):

```
Financial Statement Checklist

□ Header includes cooperative name and period
□ Revenue section complete
□ Expense section itemized
□ Net surplus/deficit calculated correctly
□ Notes explain significant items
□ Format follows CSEA template
□ Signed by treasurer
□ Reviewed by auditor

Score: ___/8 = ___%
```

## Pre/Post Testing

### Design Principles

**Coverage**: Test should cover all major learning objectives

**Parallelism**: Pre and post tests should be equivalent
- Same difficulty level
- Same content coverage
- Different specific questions (to prevent memorization)

**Length**: 15-25 questions typical for knowledge assessment

### Sample Pre/Post Test Structure

```
Part 1: Knowledge (10 questions)
- 5 MC questions on key concepts
- 3 T/F questions on facts
- 2 fill-in-blank on terminology

Part 2: Application (5 questions)
- 2 calculation problems
- 2 scenario analysis
- 1 case-based question

Part 3: Analysis (5 questions)
- 2 compare/contrast questions
- 2 evaluation questions
- 1 recommendation question
```

### Analyzing Results

**Learning Gain Calculation**:
```
Learning Gain = (Post-test - Pre-test) ÷ (Max Score - Pre-test) × 100

Example:
Pre-test: 45/100
Post-test: 85/100
Max: 100

Gain = (85 - 45) ÷ (100 - 45) × 100 = 40/55 × 100 = 73%
```

**Interpreting Results**:
| Gain | Interpretation |
|------|----------------|
| >70% | High learning gain - excellent |
| 50-70% | Medium gain - good |
| 30-50% | Low gain - needs improvement |
| <30% | Minimal gain - redesign needed |

## MoroAcademy Assessment Data

### Quiz Data Structure

```typescript
// Quiz stored in Lesson.quiz_data
{
  questions: [
    {
      id: string;
      type: 'multiple_choice' | 'true_false' | 'short_answer';
      question: string;
      options?: string[];  // For MC
      correct_answer: string | number;  // Answer or index
      explanation?: string;  // Feedback text
      points: number;
    }
  ],
  passing_score: number;  // Percentage, e.g., 70
  time_limit_minutes?: number;
  max_attempts?: number;
}
```

### Workshop Assessment Structure

```typescript
// WorkshopAssessment model fields
{
  assessment_type: 'PRE' | 'POST' | 'SELF' | 'QUIZ' | 'OUTPUT';
  title: string;
  description: string;
  duration_minutes: number;
  questions: QuestionObject[];
  output_requirements: OutputRequirement[];
  passing_score: number;  // Percentage
  max_score?: number;
  is_required: boolean;
}
```

### Question Object Format

```typescript
interface QuestionObject {
  id: string;
  type: 'mc' | 'tf' | 'fill' | 'match' | 'short' | 'essay';
  text: string;
  points: number;
  options?: {
    id: string;
    text: string;
    is_correct?: boolean;
  }[];
  correct_answer?: string;
  rubric?: string;  // For essay/short answer
  feedback?: {
    correct: string;
    incorrect: string;
  };
}
```

### Output Requirement Format

```typescript
interface OutputRequirement {
  id: string;
  title: string;
  description: string;
  criteria: {
    id: string;
    description: string;
    points: number;
    levels: {
      score: number;
      description: string;
    }[];
  }[];
  max_points: number;
}
```

## Assessment Best Practices

### Do's
- Align every question to an objective
- Use clear, simple language
- Provide immediate feedback when possible
- Mix question types for variety
- Test before deploying
- Review analytics regularly

### Don'ts
- Don't trick learners with confusing wording
- Don't test on content not covered
- Don't make assessments too long
- Don't rely solely on recall questions
- Don't ignore assessment data
- Don't use assessments punitively

### Quality Checklist

Before publishing any assessment:
- [ ] Every item linked to learning objective
- [ ] Instructions are clear
- [ ] Questions are unambiguous
- [ ] Correct answers verified
- [ ] Distractors are plausible
- [ ] Time limit is reasonable
- [ ] Scoring criteria defined
- [ ] Feedback provided
- [ ] Tested by someone else
