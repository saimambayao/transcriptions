## Facilitation Fundamentals

### Role of the Facilitator

**Facilitator vs Presenter**:
| Presenter | Facilitator |
|-----------|-------------|
| Delivers content | Guides discovery |
| Center of attention | Participants are focus |
| One-way communication | Two-way dialogue |
| Expert providing answers | Process guide asking questions |

**Core Competencies**:
- Active listening
- Question asking
- Time management
- Conflict resolution
- Energy reading
- Flexible adaptation

### Room Setup Configurations

```
U-Shape (best for 15-25):
  ┌─────────────────┐
  │   Facilitator   │
```

## Session Design Patterns

### Watch-Try-Coach (20 min total)

**Structure**:
```
Watch (5 min): Facilitator demonstrates skill/concept
Try (5 min): Individual practice attempt
Coach (10 min): Pairs provide feedback to each other
```

**When to Use**: Skill-building, procedures, techniques
**Example**: Calculating patronage refunds

### Think-Pair-Share (15 min total)

**Structure**:
```
Think (3 min): Individual reflection on prompt
Pair (5 min): Discuss with partner
Share (7 min): Select pairs share with group
```

**When to Use**: Building on prior knowledge, problem-solving
**Example**: "What compliance challenges has your coop faced?"

### Gallery Walk (30-45 min total)

**Structure**:
```
Create (15 min): Teams create poster/output
Post (2 min): Display around room
Walk (15 min): Rotate, review others' work, add comments
Debrief (10 min): Synthesis discussion
```

**When to Use**: Sharing diverse perspectives, peer learning
**Example**: Each team presents their business model canvas

### Jigsaw Learning (45-60 min)

**Structure**:
```
Expert Groups (20 min): Each group masters one topic
Home Groups (20 min): Mix members to teach each other
Synthesis (10 min): Full group Q&A
```

**When to Use**: Covering multiple topics efficiently
**Example**: Different coop types (agricultural, consumer, producer)

### Case Study Analysis (45-60 min)

**Structure**:
```
Read (10 min): Individual case review
Analyze (15 min): Small group discussion
Decide (10 min): Group decision/recommendation
Present (15 min): Share and defend decisions
Debrief (10 min): Key lessons
```

**When to Use**: Applying concepts to real situations
**Example**: Analyzing a cooperative governance conflict

## Engagement Techniques

### Icebreakers and Energizers

**Opening Icebreakers** (5-10 min):
- One-word check-in: Current mood/expectation
- Two Truths and a Lie: Fun introductions
- Common Ground: Find shared experiences
- Expectation cards: What participants hope to learn

**Mid-session Energizers** (2-5 min):
- Stand and stretch
- Quick partner share
- Movement activity
- Brain teaser
- Song or chant

### Questioning Techniques

**Question Types**:
```
Opening: "What comes to mind when you hear 'cooperative governance'?"
Probing: "Can you tell me more about that?"
Clarifying: "Are you saying that...?"
Redirecting: "What do others think about that?"
Summarizing: "So the key points are...?"
Closing: "What's one thing you'll do differently?"
```

**Wait Time**: Pause 5-7 seconds after asking a question

### Managing Participation

**Encouraging Quiet Participants**:
- Direct questions: "Maria, what's your perspective?"
- Small group first: Let them practice before large group
- Written input: Use cards or sticky notes
- Pair discussions: Lower-stakes sharing

**Managing Dominant Participants**:
- Redirect: "Thanks, let's hear from others"
- Structure: "One comment per person this round"
- Private conversation: Enlist their help in encouraging others
- Written responses: Everyone contributes equally

### Handling Difficult Situations

| Situation | Response |
|-----------|----------|
| Off-topic discussion | "Great point. Let's capture that for later..." |
| Conflict between participants | "You both have valid perspectives. Let's look at..." |
| Technical failure | Have backup activity ready; embrace the pause |
| Energy slump (post-lunch) | Energizer, movement, or change of pace |
| Confused participants | Check understanding, revisit with different approach |
| Time running short | Prioritize, adjust, communicate changes |

## Multi-Day Program Design

### Day Flow Template

**Day 1: Foundation**
```
8:00-8:30   Registration and welcome
8:30-9:00   Icebreakers, expectations setting
9:00-10:30  Module 1: Foundations
10:30-10:45 Break
10:45-12:00 Module 2: Core Concepts
12:00-1:00  Lunch
1:00-2:30   Module 3: Application Workshop
2:30-2:45   Break
2:45-4:00   Module 4: Team Activity
4:00-4:30   Day 1 Synthesis and preview
```

**Middle Days: Deep Dive**
```
8:00-8:30   Recap and warm-up
8:30-10:30  Advanced content
10:30-10:45 Break
10:45-12:00 Workshop/simulation
12:00-1:00  Lunch
1:00-3:00   Extended practice/project work
3:00-3:15   Break
3:15-4:30   Peer learning/presentation
```

**Final Day: Integration**
```
8:00-8:30   Recap and final preparation
8:30-10:30  Completing final outputs
10:30-10:45 Break
10:45-12:00 Presentations/gallery walk
12:00-1:00  Lunch
1:00-2:00   Post-assessment
2:00-3:00   Synthesis and action planning
3:00-4:00   Certification ceremony and closing
```

### Program Pacing

**Energy Curve Management**:
```
Morning:   ████████████ High energy - complex content
Post-lunch: ████░░░░░░░░ Low energy - interactive activities
Afternoon:  █████████░░░ Moderate - hands-on work
End of day: ██████░░░░░░ Winding down - synthesis
```

**Content Distribution**:
- 60% new content
- 20% practice/application
- 10% discussion/reflection
- 10% assessment/feedback

## Simulation Games

### Design Elements

**Essential Components**:
```
1. Clear Objective: What participants will learn
2. Rules: How the game works
3. Roles: What each participant does
4. Resources: Starting materials/tokens
5. Decisions: Choices participants make
6. Consequences: Results of decisions
7. Rounds: Iteration cycles
8. Debrief: Learning extraction
```

### Simulation Flow

```
Introduction (10 min)
├── Explain context and objective
├── Distribute roles and materials
└── Review rules and timeline

Round 1 (15-20 min)
├── Teams make decisions
├── Execute transactions
└── Record results

Inter-round (5 min)
├── Brief reflection
└── Introduce new element/event

Rounds 2-N (15-20 min each)
├── Progressive complexity
├── New challenges/events
└── Team adaptations

Final Round
├── Final decisions
└── Calculate final scores

Debrief (20-30 min)
├── What happened?
├── Why did it happen?
├── What did you learn?
└── How does this apply to real work?
```

### MoroAcademy Simulation Types

**Cooperative Business Simulation**:
- Teams form cooperatives
- Make business decisions each round
- Experience market conditions
- Learn cooperative principles through action

**Financial Management Game**:
- Teams manage coop finances
- Make loan, investment, dividend decisions
- Experience consequences of choices
- Learn prudent financial management

### Debriefing Framework

**What? So What? Now What?**:
```
WHAT happened?
- What did you observe?
- What decisions did your team make?
- What were the outcomes?

SO WHAT does it mean?
- Why did that happen?
- What patterns emerged?
- What surprised you?

NOW WHAT will you do?
- How does this apply to your real work?
- What will you do differently?
- What's your first action step?
```

## Hybrid/Virtual Considerations

### Virtual Facilitation Adjustments

**Session Length**:
- Max 90 minutes per session
- 10-minute breaks every 45 minutes
- Full-day workshops: Split across 2 half-days

**Engagement Tools**:
- Polls (every 10-15 min)
- Chat prompts
- Breakout rooms
- Digital whiteboards (Miro, Mural)
- Reaction buttons

**Technical Setup**:
```
Required:
- Stable internet
- Webcam (facilitator on camera)
- Clear audio
- Screen sharing
- Breakout room capability

Nice to have:
- Co-facilitator for chat/tech
- Backup slide deck
- Phone backup for audio
```

### Hybrid Meeting Tips

**Room Design**:
- Camera angle shows all in-room participants
- Microphone picks up all in-room voices
- Screen visible to both in-room and remote
- Dedicated person monitors remote participants

**Inclusion Strategies**:
- Alternate between in-room and remote contributions
- Use chat for all participants (levels playing field)
- Call on remote participants by name
- Assign in-room buddy for remote participants

### Virtual Engagement Checklist

- [ ] Welcome participants as they join
- [ ] State expectations (cameras, mute protocol)
- [ ] Use names when calling on people
- [ ] Acknowledge chat contributions
- [ ] Vary interaction types frequently
- [ ] Watch for engagement signals (or lack thereof)
- [ ] Have backup activities for tech issues
- [ ] End with clear next steps

