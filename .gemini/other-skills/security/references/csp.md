# Content Security Policy (CSP) Compliance

**MANDATORY for all OBCMS code (GEMINI.md Rule 12)**

Content Security Policy (CSP) is a security standard that helps prevent XSS attacks by controlling which resources can be loaded and executed. All OBCMS frontend code must be CSP-compliant with no inline scripts or styles.

## CSP Configuration

OBCMS uses django-csp with strict CSP policies defined in settings:

```python
# settings/base.py
CSP_DEFAULT_SRC = ("'none'",)
CSP_SCRIPT_SRC = ("'self'", "'nonce-{nonce}'")
CSP_STYLE_SRC = ("'self'", "'nonce-{nonce}'")
CSP_IMG_SRC = ("'self'", "data:", "https:")
CSP_FONT_SRC = ("'self'",)
CSP_CONNECT_SRC = ("'self'",)
CSP_FRAME_ANCESTORS = ("'none'",)
CSP_BASE_URI = ("'self'",)
CSP_FORM_ACTION = ("'self'",)

# Report violations in development
CSP_REPORT_ONLY = False  # Enforce CSP
CSP_REPORT_URI = '/csp-violation-report/'  # Optional violation reporting
```

**Key Principles:**
- Default to `'none'` for maximum security
- Allow scripts/styles only from `'self'` or with valid nonce
- No `'unsafe-inline'` or `'unsafe-eval'` allowed
- Use nonces for all inline scripts and styles

## Nonce-Based Script Loading

All inline scripts must use CSP nonces. The nonce is automatically generated by django-csp middleware.

**Template Pattern:**

```django
{% load static csp %}

{# External script - no nonce needed #}
<script src="{% static 'js/app.js' %}"></script>

{# Inline script - MUST use nonce #}
<script nonce="{{ request.csp_nonce }}">
    // Your JavaScript code here
    console.log('CSP-compliant inline script');
</script>
```

**HTMX Event Handlers:**

```django
{# ❌ WRONG - Inline onclick without nonce #}
<button onclick="doSomething()">Click</button>

{# ✅ CORRECT - Use HTMX or external script #}
<button hx-post="/api/action/" hx-trigger="click">Click</button>

{# ✅ CORRECT - If inline JS needed, use nonce #}
<script nonce="{{ request.csp_nonce }}">
    document.getElementById('myBtn').addEventListener('click', function() {
        doSomething();
    });
</script>
```

## Nonce-Based Style Loading

All inline styles must use CSP nonces.

**Template Pattern:**

```django
{% load static csp %}

{# External stylesheet - no nonce needed #}
<link rel="stylesheet" href="{% static 'css/app.css' %}">

{# Inline styles - MUST use nonce #}
<style nonce="{{ request.csp_nonce }}">
    .custom-class {
        color: blue;
    }
</style>
```

**Inline Style Attributes:**

```django
{# ❌ WRONG - Inline style attribute #}
<div style="color: red;">Text</div>

{# ✅ CORRECT - Use Tailwind utility classes #}
<div class="text-red-500">Text</div>

{# ✅ CORRECT - Use external CSS class #}
<div class="custom-red-text">Text</div>
```

## HTMX CSP Patterns

HTMX is fully CSP-compliant when used correctly. Avoid inline event handlers.

### HTMX Requests

```django
{# ✅ CORRECT - Standard HTMX attributes #}
<button
    hx-post="/api/tasks/{{ task.id }}/delete/"
    hx-target="#task-{{ task.id }}"
    hx-swap="outerHTML"
    hx-confirm="Delete this task?"
    class="btn btn-danger">
    Delete
</button>
```

### HTMX with CSRF Tokens

```django
{# ✅ CORRECT - CSRF token in meta tag #}
<meta name="csrf-token" content="{{ csrf_token }}">

{# External script configures HTMX #}
<script src="{% static 'js/htmx-config.js' %}"></script>
```

**In htmx-config.js:**

```javascript
// Configure HTMX to send CSRF token
document.body.addEventListener('htmx:configRequest', function(event) {
    const csrfToken = document.querySelector('meta[name="csrf-token"]').content;
    event.detail.headers['X-CSRFToken'] = csrfToken;
});
```

### HTMX Events

```django
{# ❌ WRONG - Inline event handler #}
<div hx-get="/data" hx-on:htmx:afterSwap="alert('Done')">...</div>

{# ✅ CORRECT - External event listener #}
<div hx-get="/data" id="data-container">...</div>

<script nonce="{{ request.csp_nonce }}">
    document.getElementById('data-container').addEventListener('htmx:afterSwap', function() {
        console.log('Data loaded');
    });
</script>
```

### HTMX Response Headers

```python
# views.py
from django.http import HttpResponse

def my_view(request):
    response = HttpResponse('<div>Content</div>')
    # HTMX headers work fine with CSP
    response['HX-Trigger'] = 'dataUpdated'
    response['HX-Redirect'] = '/success/'
    return response
```

## Alpine.js CSP Patterns

Alpine.js requires CSP-compatible initialization. Never use inline `x-data` with complex objects.

### Alpine.js Configuration

```django
{# ✅ CORRECT - Alpine.js with CSP #}
<script src="{% static 'js/alpine.min.js' %}" defer></script>

{# Simple Alpine directives are CSP-safe #}
<div x-data="{ open: false }">
    <button @click="open = !open">Toggle</button>
    <div x-show="open">Content</div>
</div>
```

### Complex Alpine Components

```django
{# ❌ WRONG - Complex inline x-data #}
<div x-data="{ items: [], async loadItems() { ... } }">...</div>

{# ✅ CORRECT - Use Alpine.data() in external script #}
<div x-data="itemManager">...</div>

<script nonce="{{ request.csp_nonce }}">
    document.addEventListener('alpine:init', () => {
        Alpine.data('itemManager', () => ({
            items: [],
            async loadItems() {
                const response = await fetch('/api/items/');
                this.items = await response.json();
            }
        }));
    });
</script>
```

### Alpine with HTMX

```django
{# ✅ CORRECT - Combining Alpine and HTMX #}
<div x-data="{ loading: false }">
    <button
        @click="loading = true"
        hx-post="/api/action/"
        hx-swap="outerHTML"
        :disabled="loading">
        <span x-show="!loading">Submit</span>
        <span x-show="loading">Loading...</span>
    </button>
</div>
```

## Inline Code Elimination

Common inline code patterns and their CSP-compliant alternatives:

### Inline Event Handlers

```django
{# ❌ WRONG #}
<button onclick="deleteTask(123)">Delete</button>

{# ✅ CORRECT - Use HTMX #}
<button hx-delete="/api/tasks/123/" hx-confirm="Delete?">Delete</button>

{# ✅ CORRECT - Use data attributes + external script #}
<button class="delete-btn" data-task-id="123">Delete</button>
<script nonce="{{ request.csp_nonce }}">
    document.querySelectorAll('.delete-btn').forEach(btn => {
        btn.addEventListener('click', function() {
            const taskId = this.dataset.taskId;
            deleteTask(taskId);
        });
    });
</script>
```

### Inline Styles

```django
{# ❌ WRONG #}
<div style="background-color: {{ task.status_color }};">...</div>

{# ✅ CORRECT - Use CSS custom properties #}
<div class="task-status" style="--status-color: {{ task.status_color }};">...</div>

<style nonce="{{ request.csp_nonce }}">
    .task-status {
        background-color: var(--status-color);
    }
</style>

{# ✅ BETTER - Use Tailwind classes with Django logic #}
<div class="{% if task.status == 'pending' %}bg-yellow-100{% elif task.status == 'completed' %}bg-green-100{% endif %}">...</div>
```

### Inline Scripts in Loops

```django
{# ❌ WRONG - Script in loop #}
{% for task in tasks %}
    <script>
        console.log('Task {{ task.id }}');
    </script>
{% endfor %}

{# ✅ CORRECT - Single script after loop #}
{% for task in tasks %}
    <div class="task" data-task-id="{{ task.id }}">...</div>
{% endfor %}

<script nonce="{{ request.csp_nonce }}">
    document.querySelectorAll('.task').forEach(taskEl => {
        const taskId = taskEl.dataset.taskId;
        console.log('Task', taskId);
    });
</script>
```

## CSP Violation Debugging

When CSP violations occur, use browser DevTools to identify and fix them.

### Viewing CSP Violations

**Chrome DevTools:**
1. Open DevTools (F12)
2. Go to Console tab
3. Look for red "Refused to execute inline script" or "Refused to apply inline style" errors
4. The error message shows which resource was blocked

**Example Violation:**

```
Refused to execute inline script because it violates the following Content Security Policy directive: "script-src 'self' 'nonce-abc123'". Either the 'unsafe-inline' keyword, a hash ('sha256-...'), or a nonce ('nonce-...') is required to enable inline execution.
```

### Common CSP Violations

| Violation | Cause | Fix |
|-----------|-------|-----|
| `Refused to execute inline script` | Inline `<script>` without nonce | Add `nonce="{{ request.csp_nonce }}"` |
| `Refused to apply inline style` | Inline `<style>` without nonce | Add `nonce="{{ request.csp_nonce }}"` |
| `Refused to execute inline event handler` | `onclick`, `onload`, etc. | Use HTMX or external event listener |
| `Refused to load script` | External script from blocked domain | Add domain to `CSP_SCRIPT_SRC` |
| `Refused to apply inline style attribute` | `style="..."` attribute | Use Tailwind classes or external CSS |

### CSP Violation Reporting

```python
# views.py
from django.http import JsonResponse
from django.views.decorators.csrf import csrf_exempt
import json
import logging

logger = logging.getLogger('csp')

@csrf_exempt
def csp_violation_report(request):
    """Log CSP violations for debugging"""
    if request.method == 'POST':
        try:
            violation = json.loads(request.body)
            logger.warning(f"CSP Violation: {violation}")
        except Exception as e:
            logger.error(f"Error parsing CSP report: {e}")
    return JsonResponse({'status': 'ok'})
```

```python
# urls.py
urlpatterns = [
    path('csp-violation-report/', csp_violation_report, name='csp_violation_report'),
]
```

## Testing CSP Compliance

### Manual Testing

1. **Enable CSP in Development:**
   ```python
   # settings/development.py
   CSP_REPORT_ONLY = False  # Enforce CSP strictly
   ```

2. **Test in Browser:**
   - Open the page in Chrome/Firefox
   - Open DevTools Console
   - Check for CSP violation errors
   - Fix any violations

3. **Test HTMX Interactions:**
   - Click all buttons with HTMX attributes
   - Submit all forms
   - Verify no CSP violations in console

### Automated Testing

```python
# tests/test_csp.py
from django.test import TestCase, Client

class CSPTestCase(TestCase):
    def setUp(self):
        self.client = Client()

    def test_csp_headers_present(self):
        """Verify CSP headers are present"""
        response = self.client.get('/')
        self.assertIn('Content-Security-Policy', response.headers)

    def test_no_inline_scripts(self):
        """Verify templates don't contain inline scripts without nonces"""
        response = self.client.get('/dashboard/')
        content = response.content.decode()

        # Check for inline scripts
        if '<script>' in content:
            # Ensure they have nonces
            self.assertIn('nonce=', content)

    def test_no_inline_styles(self):
        """Verify templates don't use inline style attributes"""
        response = self.client.get('/dashboard/')
        content = response.content.decode()

        # This is a simplified check
        # In practice, you'd parse the HTML properly
        self.assertNotIn('style="', content)
```

### CSP Testing Checklist

- [ ] All inline `<script>` tags have `nonce="{{ request.csp_nonce }}"`
- [ ] All inline `<style>` tags have `nonce="{{ request.csp_nonce }}"`
- [ ] No inline event handlers (`onclick`, `onload`, etc.)
- [ ] No inline `style` attributes
- [ ] HTMX attributes used instead of inline JavaScript
- [ ] Tailwind classes used instead of inline styles
- [ ] Alpine.js components use `Alpine.data()` for complex logic
- [ ] External scripts loaded from `'self'` origin
- [ ] CSRF tokens configured in meta tags, not inline scripts
- [ ] Browser console shows no CSP violations
- [ ] All HTMX interactions work correctly
- [ ] All form submissions work correctly

## CSP Best Practices for OBCMS

1. **Default to Tailwind CSS** - Use Tailwind utility classes instead of inline styles
2. **Use HTMX for Interactions** - HTMX attributes are CSP-safe
3. **Consolidate Inline Scripts** - Group small inline scripts into larger nonce-protected blocks
4. **External JS Files** - Move reusable JavaScript to external files in `static/js/`
5. **Template Inheritance** - Define nonce-protected scripts in base templates
6. **Test Early and Often** - Test CSP compliance during development, not before deployment
7. **Monitor Violations** - Set up CSP violation reporting in production
8. **Document Exceptions** - If CSP relaxation is needed, document why and get approval

## Common Patterns

### Loading Indicators with HTMX

```django
{# ✅ CSP-compliant loading indicator #}
<button
    hx-post="/api/action/"
    hx-indicator="#spinner"
    class="btn btn-primary">
    Submit
</button>
<div id="spinner" class="htmx-indicator">
    <div class="spinner-border" role="status">
        <span class="sr-only">Loading...</span>
    </div>
</div>
```

### Dynamic Form Validation

```django
{# ✅ CSP-compliant form validation #}
<form hx-post="/api/validate/" hx-target="#errors">
    <input type="text" name="email" required>
    <div id="errors"></div>
    <button type="submit">Submit</button>
</form>

{# Validation logic in external file #}
<script src="{% static 'js/form-validation.js' %}"></script>
```

### Modal Dialogs

```django
{# ✅ CSP-compliant modal with Alpine.js #}
<div x-data="{ open: false }">
    <button @click="open = true">Open Modal</button>

    <div x-show="open" class="modal-overlay">
        <div class="modal-content">
            <h2>Modal Title</h2>
            <button @click="open = false">Close</button>
        </div>
    </div>
</div>
```
