# Product Roadmap Guide

## Now/Next/Later Framework

### Overview

A flexible, outcome-focused roadmap format that avoids false precision of date-based timelines.

```
┌─────────────────┬─────────────────┬─────────────────┐
│      NOW        │      NEXT       │      LATER      │
│   Committed     │   Confident     │    Possible     │
│   In Progress   │   Planned       │    Exploratory  │
├─────────────────┼─────────────────┼─────────────────┤
│ High detail     │ Medium detail   │ Low detail      │
│ Specific scope  │ Estimated scope │ Theme/outcome   │
│ Resource        │ Rough estimate  │ No estimate     │
│ assigned        │                 │                 │
└─────────────────┴─────────────────┴─────────────────┘
```

### Time Horizons (Flexible)

| Column | Typical Range | Certainty |
|--------|--------------|-----------|
| **Now** | 0-3 months | 90%+ committed |
| **Next** | 3-6 months | 60-80% confident |
| **Later** | 6-12 months | 30-50% possible |

## Roadmap Levels

### Theme Roadmap (Strategic)
- Annual/multi-year view
- Business outcomes, not features
- For executives and board

### Release Roadmap (Tactical)
- Quarterly view
- Feature groups
- For stakeholders and partners

### Sprint Roadmap (Operational)
- 2-4 week view
- User stories
- For development team

---

## Creating a Product Roadmap

### Step 1: Define Themes

```
THEME FORMAT:
[Verb] + [Object] + [Outcome]

Examples:
- "Increase member engagement through self-service tools"
- "Reduce compliance burden with automated reporting"
- "Expand market reach via mobile accessibility"
```

### Step 2: Map Features to Themes

| Theme | Now | Next | Later |
|-------|-----|------|-------|
| Member Engagement | Dashboard v1 | Mobile app | AI recommendations |
| Compliance | Auto-reports | Risk alerts | Predictive compliance |
| Market Reach | Responsive web | PWA | Native apps |

### Step 3: Add Success Metrics

For each theme, define:
- **Input metric**: Activity measure (features shipped)
- **Output metric**: Result measure (usage, revenue)
- **Outcome metric**: Impact measure (satisfaction, retention)

## Roadmap Communication

### For Different Audiences

| Audience | Show | Hide |
|----------|------|------|
| **Executives** | Themes, outcomes, timelines | Technical details |
| **Sales** | Features, dates, differentiators | Internal priorities |
| **Dev Team** | User stories, dependencies, estimates | Business politics |
| **Customers** | Value, timing (rough), direction | Competitive info |

### Presentation Tips

1. **Lead with outcomes** - Start with "why" before "what"
2. **Show trade-offs** - Explain what's NOT on roadmap
3. **Acknowledge uncertainty** - "Later" means possible, not promised
4. **Update regularly** - Roadmap is living document

---

## Roadmap Anti-Patterns

| Anti-Pattern | Problem | Solution |
|--------------|---------|----------|
| **Date-driven** | Creates false precision | Use Now/Next/Later |
| **Feature factory** | Output over outcomes | Lead with themes |
| **Kitchen sink** | No prioritization | Say "no" explicitly |
| **Static document** | Ignores learning | Review quarterly |
| **Secret roadmap** | Misalignment | Share appropriately |

## SE-Adapted Roadmapping

### Dual-Track Roadmap

```
IMPACT TRACK              SUSTAINABILITY TRACK
├── Beneficiary outcomes   ├── Revenue features
├── Mission milestones     ├── Cost optimization
├── Community growth       ├── Market expansion
└── Impact measurement     └── Business metrics
```

### Stakeholder Considerations

| Stakeholder | Roadmap Interest |
|-------------|-----------------|
| **Beneficiaries** | When will my problem be solved? |
| **Funders/Donors** | When will impact be demonstrated? |
| **Customers** | When will I get value? |
| **Team** | What are we building and why? |
| **Regulators** | When will compliance be met? |

## Quarterly Review Process

### Review Inputs
- Sprint retrospectives
- Customer feedback
- Usage analytics
- Market changes
- Resource changes

### Review Questions
1. What did we learn?
2. What assumptions changed?
3. What should move to NOW?
4. What should be deprioritized?
5. What new opportunities emerged?

### Review Output
- Updated roadmap
- Revised priorities
- Communicated changes
- Documented decisions
