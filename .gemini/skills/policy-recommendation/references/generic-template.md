# Generic Policy Recommendation Template

**Reference Number Format:** `[AGENCY]-PR-[YYYY]-[XXX]`
Example: `MOGESMED-PR-2026-001`, `OOBC-PR-2026-005`

---

## Complexity Tiers

| Tier | Type | Length | Use When |
|------|------|--------|----------|
| **Tier 1** | Policy Brief | 2–8 pages | Decision-maker briefing, time-sensitive, front-loaded action |
| **Tier 2** | Standard Policy Recommendation | 8–15 pages | **DEFAULT** — full analytical structure, Cabinet or Parliament submission |
| **Tier 3** | Comprehensive Policy Paper | 20+ pages | Handled by `/policy-paper` skill — not this template |

---

### EXECUTIVE SUMMARY
*Write this section last. 3–5 paragraphs covering: the problem, the recommended option, key benefits, and the ask (decision/action required). Avoid jargon.*

---

### 1. CONTEXT AND RATIONALE

#### 1.1 Problem Statement
*State the core policy problem in 2–4 sentences. Include who is affected, the scale, and why government action is needed now.*

#### 1.2 Situation Analysis
*Provide evidence-based context. Include a **Problem Tree** (causes → problem → effects) either as a diagram description or bullet structure.*

**Problem Tree (abbreviated format):**
- **Root Causes:** [systemic/institutional/resource factors]
- **Core Problem:** [the policy gap or failure]
- **Effects:** [social, economic, governance consequences]

#### 1.3 Evidence Base
*Cite data sources: government statistics, surveys, baseline studies, subnational data. Distinguish between primary and secondary sources. Note data gaps.*

#### 1.4 Strategic Alignment
*Show alignment with:*
- *BDP 2023–2028 (goal/strategy/chapter)*
- *BOL / RA 11054 (article/section)*
- *National legislation or Executive Orders*
- *International frameworks (SDGs, CEDAW, etc.) if applicable*

---

### 2. STAKEHOLDER ANALYSIS

#### 2.1 Stakeholder Mapping
*Identify all affected parties: primary beneficiaries, implementing agencies, oversight bodies, civil society, private sector.*

#### 2.2 Power-Interest Matrix

| Stakeholder | Power | Interest | Engagement Strategy |
|-------------|-------|----------|---------------------|
| [Name/Group] | High/Med/Low | High/Med/Low | [Consult / Inform / Partner / Monitor] |

#### 2.3 Consultation Summary
*Briefly describe consultations conducted (FGDs, key informant interviews, public hearings). Note key concerns raised and how the recommendation responds to them.*

---

### 3. POLICY OPTIONS AND RECOMMENDATION

#### 3.1 Policy Alternatives
*Present a minimum of 3 options, including the status quo (do nothing) as Option A.*

| Option | Description | Pros | Cons |
|--------|-------------|------|------|
| A — Status Quo | Continue current approach | [+] | [-] |
| B — [Title] | [Brief description] | [+] | [-] |
| C — [Title] | [Brief description] | [+] | [-] |

#### 3.2 Evaluation Criteria
*Evaluate each option against OECD/DAC adapted criteria + Bardach's feasibility lens:*

| Criterion | Option A | Option B | Option C |
|-----------|----------|----------|----------|
| Relevance — addresses root cause? | | | |
| Effectiveness — likely to achieve goal? | | | |
| Efficiency — cost-benefit ratio? | | | |
| Equity — benefits reach marginalized groups? | | | |
| Sustainability — viable beyond project period? | | | |
| Political Feasibility — stakeholder support? | | | |
| **Overall Rating** | | | |

#### 3.3 Recommended Option
*State the recommended option clearly. Include:*
- **Rationale** — why this option over others
- **Theory of Change narrative** — If [inputs] → then [activities] → then [outputs] → then [outcomes] → then [impact], assuming [key assumptions]
- **Logic Model summary** (Inputs / Activities / Outputs / Outcomes / Impact)

#### 4.1 Institutional Arrangements
*Identify the lead agency, co-implementers, oversight body, and coordination mechanism.*

### 5. MONITORING, EVALUATION, AND LEARNING

#### 5.1 Results Framework

| Level | Indicator | Baseline | Target | Means of Verification | Frequency |
|-------|-----------|----------|--------|----------------------|-----------|
| Impact | [Long-term change] | [Value] | [Value] | [Source] | Annual |
| Outcome | [Intermediate change] | [Value] | [Value] | [Source] | Quarterly |
| Output | [Deliverable] | [Value] | [Value] | [Source] | Monthly |

*All indicators must be SMART (Specific, Measurable, Achievable, Relevant, Time-bound).*

#### 5.2 Data Collection and Reporting
*Specify data collection methods, responsible units, reporting schedule, and report format (dashboard, narrative, Parliament brief).*

#### 5.3 Evaluation Plan
*Plan evaluations using OECD/DAC 6 criteria: Relevance, Coherence, Effectiveness, Efficiency, Impact, Sustainability. Identify evaluation type (process, midterm, summative) and timing.*

## TIER 1: POLICY BRIEF (ABBREVIATED)

*Front-loaded, action-oriented. Decision-maker reads the first page and can act.*

### HEADER
```
[Agency] | Policy Brief No.: [REF] | Date:
Subject: [One-line policy question]
Recommended Action: [One sentence — the ask]
```

### Executive Summary
*1 paragraph: problem, recommendation, and ask. Written for a non-specialist reader.*

### Problem Statement and Evidence
*2–3 bullet points: What is the problem? Who is affected? What does the data show?*

### Policy Options and Recommended Action
*Table with 2–3 options; highlight the recommended option. Include a one-line rationale.*

### Implementation Snapshot
*3–5 bullets: who does what, rough timeline, estimated cost, key risk.*

### Appendix: Key Data Tables *(optional)*
*Supporting tables that are too detailed for the body.*
