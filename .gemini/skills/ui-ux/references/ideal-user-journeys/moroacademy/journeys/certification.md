# Assessment to Certificate Journey - MoroAcademy

## Overview
This journey maps the experience of a learner going through a structured assessment process, typically for mandatory training programs like the Entrepreneurship Development Program (EDP). It covers pre-assessment, learning, post-assessment, and certificate issuance with emphasis on compliance tracking and official recognition.

## Persona
**Name**: Sarah Macarambon
**Age**: 29
**Role**: Social Enterprise Founder
**Context**: Sarah is registering her social enterprise with CSEA and learned that completing the EDP (Entrepreneurship Development Program) is required for SE registration. She needs to complete the training and obtain a valid certificate to submit with her registration documents. She has 2 weeks to complete this requirement.
**Goals**:
- Complete the mandatory EDP training
- Pass the required assessments
- Obtain an official, verifiable certificate
- Submit certificate as part of SE registration

## Journey Map

### Stage 1: Requirement Discovery
**User Goal**: Understand certification requirements and how to fulfill them

**Actions**:
1. Learns about EDP requirement during SE registration process
2. Clicks link to MoroAcademy or searches for EDP
3. Finds EDP course page with requirements information
4. Understands pre-assessment, learning, and post-assessment structure
5. Notes certificate validity period and verification process
6. Decides to enroll and complete the program

**Touchpoints**:
- SE registration page (with EDP requirement callout)
- MoroAcademy EDP landing page
- Program requirements section
- Certificate information tooltip
- Enrollment CTA

**Emotions**: Slightly stressed about requirement, motivated to complete quickly
**Pain Points**:
- Unclear what exactly is required
- Not knowing how long the program takes
- Uncertainty about certificate validity
- Fear of failing assessments

**Opportunities**:
- Clear requirement explanation with timeline
- "Required for Registration" badge on course
- Estimated completion time prominently displayed
- FAQ section addressing common concerns
- Pass rate statistics to reduce anxiety

### Stage 2: Program Enrollment
**User Goal**: Enroll in the EDP program and prepare for assessment

**Actions**:
1. Clicks "Enroll in EDP" button
2. Logs in or creates MoroAcademy account
3. Reviews program structure and assessment requirements
4. Confirms enrollment (free or paid as applicable)
5. Receives welcome message with next steps
6. Understands pre-assessment is required first

**Touchpoints**:
- EDP enrollment flow
- Account creation/login
- Program overview page
- Welcome email
- Pre-assessment introduction

**Emotions**: Committed, slightly anxious about assessments
**Pain Points**:
- Complicated enrollment process
- Unclear what happens after enrollment
- Not understanding pre-assessment purpose
- Fear of starting and failing

**Opportunities**:
- Streamlined enrollment (minimal steps)
- Clear "What's Next" guidance
- Explain pre-assessment is diagnostic, not pass/fail
- Encourage immediate start
- Show others who completed successfully

### Stage 3: Pre-Assessment
**User Goal**: Complete pre-assessment to establish baseline knowledge

**Actions**:
1. Starts pre-assessment from dashboard
2. Reads instructions (purpose, no passing requirement)
3. Answers diagnostic questions
4. Submits pre-assessment
5. Reviews results showing knowledge gaps
6. Sees personalized learning recommendations
7. Proceeds to learning modules

**Touchpoints**:
- Pre-assessment introduction page
- Assessment question interface
- Results dashboard
- Knowledge gap analysis
- Recommended modules highlight

**Emotions**: Nervous initially, relieved it's diagnostic, curious about gaps
**Pain Points**:
- Thinking pre-assessment affects final score
- No clear feedback on answers
- Not understanding why it's required
- Long assessment causing fatigue

**Opportunities**:
- Clear "This is diagnostic only" messaging
- Estimated time for assessment
- Progress indicator during assessment
- Visual knowledge gap chart
- Personalized "Focus on these areas" guidance
- Comparison: before vs. after (teaser)

### Stage 4: Structured Learning
**User Goal**: Complete learning modules to build required competencies

**Actions**:
1. Enters learning phase with modules organized by competency
2. Completes required modules in sequence
3. Takes module quizzes to check understanding
4. Tracks progress toward completion
5. Reviews supplementary materials and resources
6. Completes all required modules

**Touchpoints**:
- Course player with module navigation
- Video and text lessons
- Module quizzes
- Progress dashboard
- Resource downloads (templates, guides)
- Module completion indicators

**Emotions**: Engaged, occasionally overwhelmed, satisfied as completing
**Pain Points**:
- Mandatory sequence feeling restrictive
- Too much content to absorb
- Module quizzes too difficult
- Unclear which content is essential
- Time pressure from deadline

**Opportunities**:
- Clear learning path visualization
- "Essential" vs "Supplementary" content tags
- Quiz retakes with feedback
- Progress relative to deadline indicator
- "Catch up" mode for time-pressed learners
- Practical examples from Bangsamoro context

### Stage 5: Post-Assessment Preparation
**User Goal**: Prepare for final assessment with confidence

**Actions**:
1. Completes all learning modules
2. Reviews summary of key concepts
3. Accesses practice questions or review materials
4. Checks assessment format and requirements
5. Reviews passing score threshold
6. Feels ready to take final assessment

**Touchpoints**:
- Course completion notification (pre-assessment phase)
- Review materials section
- Practice quiz (optional)
- Assessment guidelines page
- "Start Final Assessment" gate

**Emotions**: Anticipation, nervousness, desire to succeed
**Pain Points**:
- No practice questions available
- Unclear what will be tested
- Anxiety about passing threshold
- No review of previous quiz mistakes

**Opportunities**:
- Comprehensive review summary
- Practice assessment with similar format
- "Review Your Weak Areas" tool
- Clear passing requirements explanation
- Encouragement based on quiz performance
- "You're ready" confidence indicator

### Stage 6: Post-Assessment (Final Assessment)
**User Goal**: Pass the final assessment to earn certificate

**Actions**:
1. Initiates final assessment
2. Reads rules (time limit, attempt policy, passing score)
3. Answers assessment questions
4. Reviews answers before submission
5. Submits final assessment
6. Receives immediate score and pass/fail result
7. If failed, understands retake policy

**Touchpoints**:
- Assessment start page with rules
- Timed assessment interface
- Question navigation panel
- Review answers screen
- Submit confirmation
- Results page

**Emotions**: Focused, stressed, relieved or disappointed at result
**Pain Points**:
- Time pressure causing errors
- Can't review answers before submit
- Unclear scoring breakdown
- No feedback on wrong answers
- Harsh retake policy

**Opportunities**:
- Clear timer display (not anxiety-inducing)
- Answer review before final submission
- Detailed score breakdown by topic
- Feedback on incorrect answers
- Reasonable retake policy (2-3 attempts)
- If failed, specific study recommendations

### Stage 7: Certificate Issuance
**User Goal**: Receive official, verifiable certificate

**Actions**:
1. Views congratulations/pass confirmation
2. Reviews certificate details (name, date, validity)
3. Generates and downloads PDF certificate
4. Notes certificate verification URL/QR code
5. Checks certificate in "My Certificates" section
6. Shares certificate or submits for SE registration

**Touchpoints**:
- Pass confirmation page
- Certificate preview
- Download button (PDF)
- Certificate verification page
- My Certificates dashboard
- Share options

**Emotions**: Proud, accomplished, relieved, ready to use certificate
**Pain Points**:
- Certificate takes time to generate
- Certificate design looks unofficial
- Can't correct name errors easily
- Verification process unclear
- Certificate not immediately usable

**Opportunities**:
- Instant certificate generation
- Professional, branded certificate design
- CSEA/Government logos for legitimacy
- Clear validity period display
- QR code for instant verification
- Direct integration with SE registration
- LinkedIn/social sharing option

### Stage 8: Certificate Verification
**User Goal**: Verify certificate authenticity for official use

**Actions**:
1. Shares certificate with CSEA registration
2. CSEA staff scans QR code or enters certificate ID
3. Verification page confirms authenticity
4. Shows certificate holder, program, date, validity
5. Certificate accepted for registration

**Touchpoints**:
- Certificate with QR code
- Verification webpage (/academy/verify/[id])
- Verification result display
- SE registration integration

**Emotions**: Confident in certificate legitimacy
**Pain Points**:
- Verification page not working
- Details don't match records
- Expired certificate without notice
- Manual verification too slow

**Opportunities**:
- Reliable verification system
- Clear verification result display
- Pre-expiry notifications
- Direct API integration with CSEA systems
- Bulk verification for CSEA staff

## Success Metrics
- **Assessment Completion Rate**: % who complete pre and post assessments
- **First-Attempt Pass Rate**: % who pass post-assessment on first try
- **Average Score**: Mean score on post-assessment
- **Time to Certificate**: Days from enrollment to certificate
- **Certificate Verification Usage**: Number of verifications performed
- **Registration Conversion**: % who complete SE registration after certificate

**Types** (under `frontend/src/lib/types/academy/...`):

- `assessment.ts` - Assessment, Question, Answer, Attempt types
- `certificate.ts` - Certificate, Verification, QRCode types

### Phase 3: Backend Integration
**Django Ninja Endpoints** (in `backend/apps/training/api.py`):
- `GET /api/training/programs/{slug}` - Get program details with requirements
- `POST /api/training/programs/{slug}/enroll` - Enroll in mandatory program
- `GET /api/training/assessments/{id}` - Get assessment questions
- `POST /api/training/assessments/{id}/submit` - Submit assessment answers
- `GET /api/training/assessments/{id}/results` - Get assessment results
- `GET /api/training/certificates` - List user's certificates
- `GET /api/training/certificates/{id}` - Get certificate details
- `POST /api/training/certificates/{enrollmentId}/generate` - Generate certificate
- `GET /api/training/certificates/{id}/download` - Download certificate PDF
- `GET /api/training/verify/{verificationId}` - Public verification endpoint

**Schemas** (in `backend/apps/training/schemas.py`):
- `ProgramSchema` - Program with requirements and timeline
- `AssessmentSchema` - Assessment with questions (without answers)
- `AssessmentSubmissionSchema` - Submitted answers payload
- `AssessmentResultSchema` - Results with score breakdown
- `CertificateListSchema` - Certificate summary for listing
- `CertificateDetailSchema` - Full certificate with verification data
- `VerificationSchema` - Public verification result
