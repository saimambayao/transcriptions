# Provider Onboarding Journey - MoroAcademy

## Overview
This journey maps the experience of a training provider (cooperative, social enterprise, or Training Service Provider) applying to become a verified provider on MoroAcademy, completing verification, setting up their branding, creating their first course, and accessing analytics. This journey ensures quality control while making onboarding accessible.

## Persona
**Name**: BACOMSEDECO Training Office
**Organization Type**: Cooperative Training Department
**Context**: Bangsamoro Cooperative Development and Multipurpose Cooperative (BACOMSEDECO) is a large federation of cooperatives with an established training department. They have been conducting in-person trainings for member cooperatives for years. The Training Officer, Ustaz Ibrahim, wants to register BACOMSEDECO as a training provider on MoroAcademy to reach more cooperatives digitally and offer their expertise to the wider Bangsamoro cooperative community.
**Goals**:
- Register BACOMSEDECO as an official training provider
- Set up branded presence on MoroAcademy
- Create and publish first online course
- Track learner enrollments and completion

## Journey Map

### Stage 1: Provider Discovery
**User Goal**: Understand how to become a training provider on MoroAcademy

**Actions**:
1. Learns about MoroAcademy provider program
2. Visits "Become a Provider" or "For Trainers" page
3. Reviews provider types and benefits
4. Understands requirements for different provider tiers
5. Reviews example provider profiles
6. Decides to apply as cooperative provider

**Touchpoints**:
- For Trainers/Become a Provider landing page
- Provider types comparison
- Benefits section
- Requirements checklist
- Example profiles
- Apply CTA

**Emotions**: Interested, evaluating opportunity
**Pain Points**:
- Hard to find information about becoming a provider
- Unclear provider types and differences
- Requirements not clearly stated
- No examples of successful providers
- Unclear value proposition

**Opportunities**:
- Dedicated "Become a Provider" page
- Clear provider tier comparison (Cooperative, SE, TSP)
- Benefits by tier (revenue share, branding, analytics)
- Success stories from existing providers
- Provider application CTA prominent
- FAQ for potential providers

### Stage 2: Provider Application
**User Goal**: Submit application to become a verified provider

**Actions**:
1. Clicks "Apply as Provider" button
2. Selects provider type (Cooperative)
3. Creates MoroAcademy organizational account
4. Fills organization details (name, CIN, contact)
5. Uploads verification documents (COR, CDO)
6. Describes training expertise and capacity
7. Submits application

**Touchpoints**:
- Provider type selection
- Organization registration form
- Document upload interface
- Expertise description form
- Application submission confirmation

**Emotions**: Committed, hopeful, thorough
**Pain Points**:
- Long, complicated application form
- Unclear which documents are required
- Upload failures or size limits
- No progress saving
- No application status tracking

**Opportunities**:
- Multi-step application wizard
- Clear document requirements with examples
- Reliable upload with progress indicator
- Auto-save progress
- Application tracking dashboard
- Clear timeline for review

### Stage 3: Verification Review
**User Goal**: Get application reviewed and approved

**Actions**:
1. Receives application confirmation email
2. Checks application status in dashboard
3. Responds to any clarification requests
4. Provides additional documents if needed
5. Waits for verification decision
6. Receives approval notification

**Touchpoints**:
- Confirmation email
- Application status dashboard
- Message center for clarifications
- Additional document upload
- Approval notification

**Emotions**: Waiting anxiously, responsive to requests
**Pain Points**:
- No visibility into review progress
- Long wait without updates
- Rejection without clear reasons
- No way to communicate with reviewer
- Re-application process unclear

**Opportunities**:
- Real-time application status tracking
- Estimated review timeline
- Clear communication channel for clarifications
- Specific feedback if changes needed
- Appeal or resubmission process
- Expedited review for CSEA-registered entities

2. Logs into provider dashboard for first time

5. Understands revenue model and payment setup
6. Sets up payment/payout information

**Touchpoints**:

- Provider dashboard
- Onboarding tutorial/tour
- Provider handbook
- Revenue/payment settings
- Help resources

**Emotions**: Excited, learning the system, slightly overwhelmed
**Pain Points**:
- No guidance after approval
- Dashboard confusing
- Unclear revenue model
- No handbook or documentation
- Payment setup complicated

**Opportunities**:
- Automated onboarding sequence
- Interactive dashboard tour
- Video tutorials for key features
- Provider handbook PDF
- Clear revenue model explanation
- Simple payment/payout setup
- Dedicated support for new providers
