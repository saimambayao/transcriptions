### When Invoked by /social-enterprise

**Context**: You are helping someone develop a social enterprise. `/social-enterprise` has invoked you for Phase 3: DETERMINE & DIRECT Business Impact.

**Inputs from /lean-canvas:**

| Input | Description | Maps To |
|-------|-------------|---------|
| Problem + Solution | The SE's intervention | ToC Interventions/Activities |
| Key Metrics | Success measures | Outcome Indicators |
| UVP | Value proposition | Impact statement basis |
| Customer Segments | Beneficiaries + customers | Target Groups |

**Your Outputs Must Include:**

| Output | Description | Feedback To |
|--------|-------------|-------------|
| **Theory of Change** | Complete impact pathway | SE implementation |
| **Assumptions** | What must be true | `/design-thinking` (test) |
| **Indicators** | Measurement framework | `/lean-canvas` (key metrics) |
| **Impact Statements** | Long-term outcomes | `/lean-canvas` (unfair advantage) |

**Workflow for SE Impact Planning:**

```text
SE IMPACT PLANNING WORKFLOW:
│
├── 1. Receive from /lean-canvas
│   ├── Problem + Solution → Interventions
│   ├── Key Metrics → Indicator development
│   ├── UVP → Impact goal framing
│   └── Customer Segments → Target groups
│
├── 2. Build ToC (Backward Mapping)
│   ├── Define long-term impact (from UVP)
│   ├── Map outcomes pathway (LTO → MTO → STO)
│   ├── Identify assumptions at each link
│   ├── Develop indicators for each outcome
│   └── Connect interventions (from Solution)
│
├── 3. Validate & Refine
│   ├── Test assumptions
│   ├── Check plausibility
│   └── Ensure measurability
│
└── FEEDBACK OUTPUTS:
    ├── Assumptions to test → /design-thinking
    ├── Impact goals → /lean-canvas (unfair advantage)
    └── Indicator alignment → /lean-canvas (key metrics)
```
