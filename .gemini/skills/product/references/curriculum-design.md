# Product Management Curriculum Design

For use with `/academy` skill when designing PM courses and workshops.

## Competency Framework

### Foundational Level

| Competency | Learning Objectives |
|------------|-------------------|
| **PM Fundamentals** | Define product management role, explain PM lifecycle |
| **User Understanding** | Conduct user interviews, create personas |
| **Problem Framing** | Write problem statements, identify root causes |
| **Basic Prioritization** | Apply MoSCoW, create simple backlog |

### Intermediate Level

| Competency | Learning Objectives |
|------------|-------------------|
| **Roadmapping** | Create Now/Next/Later roadmap, communicate effectively |
| **Prioritization Frameworks** | Apply RICE, Kano, make trade-offs |
| **MVP Definition** | Scope MVP, define success criteria |
| **Metrics & Analytics** | Define KPIs, interpret data |

### Advanced Level

| Competency | Learning Objectives |
|------------|-------------------|
| **Product Strategy** | Develop product vision, competitive positioning |
| **Stakeholder Management** | Navigate politics, build consensus |
| **Product-Led Growth** | Design growth loops, optimize funnels |
| **Platform Thinking** | Design ecosystems, network effects |

## Standard Course Modules

### Module 1: Introduction to Product Management

**Learning Objectives:**
- Define product management and PM role
- Explain PM in organizational context
- Identify PM career paths

**Topics:**
- What is a product?
- PM vs Project Manager vs Program Manager
- PM responsibilities and skills
- Day in the life of a PM

**Activities:**
- PM role comparison exercise
- Self-assessment of PM skills

---

### Module 2: Product Vision and Strategy

**Learning Objectives:**
- Create product vision statement
- Develop product strategy
- Align product with business goals

**Topics:**
- Vision vs Strategy vs Tactics
- Product vision statement formula
- Strategic frameworks (Porter, Blue Ocean)
- OKRs for product

**Activities:**
- Vision statement workshop
- Strategy canvas exercise

---

### Module 3: Understanding Users

**Learning Objectives:**
- Conduct user research
- Create user personas
- Map user journeys

**Topics:**
- User research methods
- Jobs to Be Done framework
- Persona development
- Journey mapping

**Activities:**
- Mock user interview
- Persona creation exercise
- Journey mapping workshop

**Link to:** `/design-thinking` for deeper user research

---

### Module 4: Feature Prioritization

**Learning Objectives:**
- Apply RICE scoring
- Use MoSCoW categorization
- Apply Kano model

**Topics:**
- Why prioritization matters
- RICE framework (SE-adapted)
- MoSCoW method
- Kano model
- Saying "no" effectively

**Activities:**
- RICE scoring exercise
- MoSCoW prioritization workshop
- Kano questionnaire practice

---

### Module 5: Roadmap Development

**Learning Objectives:**
- Create Now/Next/Later roadmap
- Communicate roadmap to stakeholders
- Update roadmap based on learnings

**Topics:**
- Roadmap types and purposes
- Now/Next/Later framework
- Theme-based roadmapping
- Roadmap communication

**Activities:**
- Roadmap creation workshop
- Stakeholder presentation practice

---

### Module 6: MVP and Lean Product

**Learning Objectives:**
- Define minimum viable product
- Scope MVP appropriately
- Design validation experiments

**Topics:**
- What is MVP (and what it's not)
- MVP types (Wizard of Oz, Concierge, etc.)
- Lean experimentation
- Build-Measure-Learn loop

**Activities:**
- MVP scoping exercise
- Experiment design workshop

---

### Module 7: Quality and Delivery

**Learning Objectives:**
- Write acceptance criteria
- Define "done" for features
- Collaborate with development teams

**Topics:**
- User stories and acceptance criteria
- Definition of Done
- Working with developers
- QA and testing basics

**Activities:**
- User story writing practice
- Acceptance criteria workshop

---

### Module 8: Product Metrics and Iteration

**Learning Objectives:**
- Define product metrics
- Analyze product data
- Make data-driven decisions

**Topics:**
- AARRR metrics (Pirate Metrics)
- North Star Metric
- A/B testing basics
- Iteration and pivoting

**Activities:**
- Metrics definition exercise
- Case study analysis

**Link to:** `/toc` for impact measurement alignment

## Assessment Methods

| Assessment | When | What It Measures |
|------------|------|------------------|
| **Roadmap Project** | Module 5 | Ability to create and present roadmap |
| **Prioritization Exercise** | Module 4 | Framework application skills |
| **MVP Pitch** | Module 6 | Scoping and validation design |
| **Final Portfolio** | End | Comprehensive PM skills |

## Workshop Formats

### Half-Day Workshop (4 hours)
- Focus: Single topic deep dive
- Example: "Feature Prioritization Masterclass"
- Format: Presentation (1hr) + Exercise (2hr) + Debrief (1hr)

### Full-Day Workshop (8 hours)
- Focus: End-to-end workflow
- Example: "From Idea to Roadmap"
- Format: Multiple modules with exercises

### Multi-Day Program (2-3 days)
- Focus: Comprehensive PM training
- Example: "Product Management Fundamentals"
- Format: All 8 modules with capstone project
