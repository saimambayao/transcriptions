# Technical Writing Standards

Plain language principles for professional documents.

## Core Rules

1. **One idea per sentence** — split compound sentences
2. **Active voice by default** — "The committee approved" not "It was approved by the committee"
3. **Specific over vague** — "15 barangays" not "several localities"
4. **Concrete over aspirational** — measurable outcomes, not hopes
5. **Action verbs** — "The ministry will implement" not "implementation will occur"

## Concision

| Remove | Replace With |
|--------|-------------|
| "in order to" | "to" |
| "at this point in time" | "now" |
| "due to the fact that" | "because" |
| "in the event that" | "if" |
| "prior to" | "before" |
| "subsequent to" | "after" |
| "with regard to" | "about" |
| "it is important to note that" | (delete) |
| "it should be noted that" | (delete) |
| "advance planning" | "planning" |
| "end result" | "result" |
| "future plans" | "plans" |
| "past history" | "history" |
| "basic fundamentals" | "fundamentals" |

## Document Structure

- **Headers**: descriptive, not generic ("Budget Allocation by Province" not "Overview")
- **Paragraphs**: 3-5 sentences max. One topic per paragraph.
- **Lists**: bullets for sets, numbers for sequences. Not both.
- **Tables**: use when comparing 3+ items across 2+ dimensions

## Readability Targets

| Audience | Grade Level | Sentence Length |
|----------|------------|-----------------|
| General public | 8-10 | 15-20 words avg |
| Professional | 10-12 | 20-25 words avg |
| Technical | 12-14 | 20-30 words avg |
| Legal | N/A | Precision over readability |

## Government Writing Conventions (Philippine)

- Spell out numbers 1-10, use digits for 11+
- Use "Republic Act No. XXXX" on first mention, "RA XXXX" thereafter
- Dates: "23 March 2026" (government style) or "March 23, 2026" (correspondence)
- Currency: "PHP 1,000,000" or "P1,000,000" (not "1M pesos")
- Agencies: spell out on first mention with acronym, acronym thereafter
