## Design Thinking Curriculum

### Framework Foundation

All Design Thinking curriculum in this folder is based on the **Stanford d.school 5-Phase Model**:

```
EMPATHIZE → DEFINE → IDEATE → PROTOTYPE → TEST
```

References from: `/Users/saidamenmambayao/apps/csea/.gemini/skills/design-thinking/`

### Course vs Workshop: When to Use

| Choose DT-101 (Course) when... | Choose DT-500 (Workshop) when... |

## DT-101: Design Thinking Fundamentals

**Type**: Online Self-Paced Course
**Duration**: 12 hours (across 7 modules)
**Difficulty**: Beginner to Intermediate

### Course Structure

| Module | Title | Duration |
|--------|-------|----------|
| 1 | Introduction to Design Thinking | 1.5 hours |
| 2 | Empathize: Understanding Users | 2 hours |
| 3 | Define: Framing the Problem | 2 hours |
| 4 | Ideate: Generating Solutions | 2 hours |
| 5 | Prototype: Building to Learn | 2 hours |
| 6 | Test: Learning from Users | 1.5 hours |
| 7 | Integration & Application | 1 hour |

### Target Learners

- Aspiring entrepreneurs validating business ideas
- Social enterprise founders seeking user-centered approaches
- Cooperative members developing new products/services
- Startup team members building innovation capabilities

### Key Features

- Video lessons (3-15 min each)
- Practical exercises with templates
- Module quizzes and final assessment
- Certificate upon completion (70%+ score)
- BARMM-localized examples and case studies

## DT-500: Design Thinking Sprint Workshop

**Type**: Live Facilitated Workshop
**Duration**: 3 days (24 hours total)
**Difficulty**: Intermediate to Advanced

### Workshop Structure

| Day | Theme | Focus |
|-----|-------|-------|
| Day 1 | Empathize & Define | User research, problem framing |
| Day 2 | Ideate & Prototype | Creative solutions, rapid prototyping |
| Day 3 | Test & Iterate | User testing, iteration, action planning |

### Target Participants

- Social enterprise managers
- Cooperative business development officers
- Startup founding teams
- Innovation champions in organizations

### Key Features

- ENGAGE methodology applied throughout
- 30/70 rule (max 30% lecture, min 70% participation)
- 5 complete output templates included
- Comprehensive facilitator guide
- Real project work with actual users

### Workshop Outputs

Participants leave with:
1. Validated Empathy Map for their target users
2. HMW statements and prioritized opportunities
3. Tested prototype of their solution
4. User feedback synthesis
5. 30-60-90 day action plan

## Methodology Applied

### ENGAGE Framework (for Workshops)

| Element | Application |
|---------|-------------|
| **E**xperience First | Activities before concepts |
| **N**arrative Thread | Story-based learning journey |
| **G**uided Practice | Structured exercises with templates |
| **A**ctive Processing | Reflection and synthesis |
| **G**roup Learning | Peer collaboration and feedback |
| **E**xtend Beyond | Action plans for post-workshop application |

### Microlearning Architecture (for Courses)

- Videos: 3-15 minutes maximum
- One concept per lesson
- Immediate application opportunities
- Progressive skill building

## BARMM Contextualization

All templates follow the **General → BARMM** two-layer approach:

1. **Layer 1 (General)**: Universal design thinking principles
2. **Layer 2 (BARMM)**: Local examples, cultural considerations, regional challenges

### Local Application Areas

- **Cooperatives**: Member-centered service design
- **Social Enterprises**: Impact-focused product development
- **Agriculture**: Farmer-centered solutions (MAFAR alignment)
- **Trade**: Consumer-driven market approaches (MTIT alignment)
- **Halal Products**: User research for halal compliance
