## Design Frameworks

### 1. ADDIE Model

The foundational instructional design framework. See [addie-framework.md](addie-framework.md) for detailed guidance.

```
ANALYZE → DESIGN → DEVELOP → IMPLEMENT → EVALUATE
   ↑                                          ↓
   └──────────── Continuous Improvement ──────┘
```

**Best for**: Traditional courses, structured programs, government/compliance training.

### 2. Understanding by Design (UbD) / Backward Design

Start with the end in mind. Design from learning outcomes backward to activities.

```
STAGE 1: IDENTIFY DESIRED RESULTS
├── What should learners know and be able to do?
├── What are the big ideas and essential questions?
└── What transfer goals do we seek?

STAGE 2: DETERMINE ACCEPTABLE EVIDENCE
├── How will we know if learners have achieved results?
├── What performance tasks show understanding?
└── What criteria and rubrics will we use?

STAGE 3: PLAN LEARNING EXPERIENCES
├── What activities will lead to desired results?
├── What sequence of instruction?
└── What resources and materials?
```

**Key Concepts**:

| Concept | Description | Example |
|---------|-------------|---------|
| **Enduring Understandings** | Big ideas that remain after details fade | "Financial transparency builds member trust" |
| **Essential Questions** | Open questions that guide inquiry | "How do cooperatives balance member needs with sustainability?" |
| **Transfer Goals** | Real-world application of learning | "Apply FRAMES compliance in daily operations" |

**Best for**: Competency-based programs, professional development, skills training.

### 3. SAM (Successive Approximation Model)

Agile, iterative approach. Build-test-refine in rapid cycles.

```
┌─────────────────────────────────────────────────────────────────┐
│  PREPARATION PHASE                                              │
│  └── Information Gathering → Savvy Start (kickoff session)     │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  ITERATIVE DESIGN PHASE                                         │
│  Design → Prototype → Review → Design → Prototype → Review...  │
│  (3 iterations minimum)                                         │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  ITERATIVE DEVELOPMENT PHASE                                    │
│  Develop → Implement → Evaluate → Develop → Implement...       │
│  (3 iterations minimum)                                         │
└─────────────────────────────────────────────────────────────────┘
```

**SAM vs ADDIE**:

| Aspect | ADDIE | SAM |
|--------|-------|-----|
| Approach | Linear, sequential | Iterative, cyclical |
| Flexibility | Low, changes costly | High, built for change |
| Time to first output | Long | Short (prototype early) |
| Stakeholder input | At defined phases | Continuous |
| Best for | Stable requirements | Evolving needs, complex topics |

**Best for**: Digital courses, rapid development, user-tested content.

### Framework Selection Guide

| Situation | Recommended Framework |
|-----------|----------------------|
| Government/compliance training | ADDIE |
| Enterprise skill building | UbD (Backward Design) |
| Digital course development | SAM |
| Time-sensitive development | SAM |
| Complex behavioral change | UbD + ADDIE hybrid |
| Simple orientation/intro | Simplified ADDIE |

## Training Needs Assessment

Before designing curriculum, systematically identify learning gaps.

### TNA Process

```
1. IDENTIFY PERFORMANCE GAPS
   ├── What are people doing vs. what should they do?
   ├── Is it a knowledge, skill, or motivation gap?
   └── What's the business impact of the gap?

2. ANALYZE CAUSES
   ├── Lack of knowledge/skills?
   ├── Environmental barriers?
   ├── Motivation issues?
   └── Process/system problems?

3. DETERMINE IF TRAINING IS THE SOLUTION
   ├── Can training solve this gap?
   ├── What non-training interventions needed?
   └── What's the ROI of training?

4. DEFINE LEARNING REQUIREMENTS
   ├── What must learners know/do?
   ├── What proficiency level is needed?
   └── How will success be measured?
```

### TNA Methods

| Method | When to Use | Pros | Cons |
|--------|-------------|------|------|
| **Surveys** | Large audience, quantitative data | Scalable, anonymous | Low response rates |
| **Interviews** | Deep insights, key stakeholders | Rich data, context | Time-intensive |
| **Focus Groups** | Explore perspectives, consensus | Interactive, diverse views | Dominant voices bias |
| **Observation** | Task analysis, process gaps | Objective, real behavior | Observer effect |
| **Document Review** | Existing data, performance records | Non-intrusive | May be outdated |
| **Skills Testing** | Measure current competencies | Objective baseline | Creates anxiety |

### TNA Checklist

```markdown
## TNA Checklist

### Performance Gap Analysis
- [ ] Current performance documented
- [ ] Desired performance defined
- [ ] Gap clearly articulated
- [ ] Root causes analyzed (knowledge, skill, motivation, environment)

### Stakeholder Input
- [ ] Leadership interviewed
- [ ] Subject matter experts consulted
- [ ] Target learners surveyed/interviewed
- [ ] Supervisors/managers input gathered

### Training Appropriateness
- [ ] Training can address the gap (vs. job aids, process change, etc.)
- [ ] Non-training interventions identified
- [ ] Cost-benefit analysis completed

### Learning Requirements
- [ ] Knowledge requirements listed
- [ ] Skill requirements listed
- [ ] Attitude/behavior requirements listed
- [ ] Proficiency levels defined

### Constraints Identified
- [ ] Budget limitations
- [ ] Time constraints
- [ ] Technology access
- [ ] Language/literacy considerations
- [ ] Geographic/logistical factors
```

## Competency-Based Design

Design curriculum around specific competencies learners must demonstrate.

### Competency Framework Structure

```
COMPETENCY AREA (Domain)
└── COMPETENCY (Skill cluster)
    └── COMPETENCY ELEMENT (Specific behavior)
        └── PERFORMANCE CRITERIA (Observable, measurable)
```

**Example - Financial Management Competency**:

```
DOMAIN: Financial Management
└── COMPETENCY: Prepare Financial Statements
    ├── ELEMENT: Record daily transactions
    │   ├── Criteria: Accurately records all transactions within 24 hours
    │   └── Criteria: Uses correct account classifications
    ├── ELEMENT: Generate balance sheet
    │   ├── Criteria: All assets and liabilities correctly categorized
    │   └── Criteria: Equity section reflects current membership data
    └── ELEMENT: Prepare income statement
        ├── Criteria: Revenue recognition per cooperative standards
        └── Criteria: Expense matching applied correctly
```

### Proficiency Levels

| Level | Description | Indicators |
|-------|-------------|------------|
| **1 - Awareness** | Knows the concept exists | Can recognize, define |
| **2 - Knowledge** | Understands principles | Can explain, describe |
| **3 - Application** | Can perform with guidance | Can apply with support |
| **4 - Proficient** | Can perform independently | Can apply consistently |
| **5 - Expert** | Can teach and innovate | Can mentor, adapt, improve |

### Competency-Based Curriculum Template

```markdown
## [Competency Name]

**Domain**: [Competency Area]
**Target Proficiency**: Level [1-5]
**Prerequisites**: [Required prior competencies]

### Performance Criteria
At the end of this training, learners can:
1. [Observable behavior + standard + condition]
2. [Observable behavior + standard + condition]
3. [Observable behavior + standard + condition]

### Learning Pathway

| Module | Focus | Proficiency Target |
|--------|-------|-------------------|
| Module 1 | Foundations | Level 2 (Knowledge) |
| Module 2 | Guided Practice | Level 3 (Application) |
| Module 3 | Independent Practice | Level 4 (Proficient) |

### Assessment Strategy

| Proficiency Check | Method | Passing Criteria |
|-------------------|--------|------------------|
| Knowledge | Quiz | 80% correct |
| Application | Simulation | Meets rubric |
| Proficiency | Work product | Supervisor sign-off |
```

## Curriculum Mapping

Ensure alignment between outcomes, content, activities, and assessments.

### Alignment Matrix

Map learning objectives to content, activities, and assessments:

```markdown
| Learning Objective | Module | Lesson | Activity Type | Assessment |
|--------------------|--------|--------|---------------|------------|
| LO1: Define cooperative principles | M1 | L1.1 | Video + Text | Quiz |
| LO2: Apply patronage refund calculation | M2 | L2.3 | Simulation | Practical |
| LO3: Evaluate governance effectiveness | M3 | L3.2 | Case study | Rubric |
```

### Bloom's Alignment

Ensure activities match cognitive level required:

| Objective Level | Activity Types | Assessment Types |
|-----------------|----------------|------------------|
| **Remember** (define, list) | Reading, video, flashcards | Multiple choice, matching |
| **Understand** (explain, summarize) | Discussion, explanation | Short answer, diagram |
| **Apply** (calculate, demonstrate) | Practice problems, simulation | Practical exercise |
| **Analyze** (compare, differentiate) | Case study, data analysis | Essay, analysis report |
| **Evaluate** (assess, justify) | Peer review, critique | Rubric-scored project |
| **Create** (design, formulate) | Project, design challenge | Portfolio, presentation |

### Horizontal Alignment (Across Modules)

Ensure consistent depth and coverage across the curriculum:

```
Module 1     Module 2     Module 3     Module 4
[Concept A]  [Concept B]  [Concept C]  [Integration]
   ↓            ↓            ↓            ↓
 Basic        Basic        Basic       All concepts
 coverage     coverage     coverage     combined
```

### Vertical Alignment (Within Module)

Ensure progression from simple to complex:

```
Session 1.1: Introduction (Remember)
     ↓
Session 1.2: Deep Dive (Understand)
     ↓
Session 1.3: Practice (Apply)
     ↓
Session 1.4: Application (Analyze/Evaluate)
```

## Scope & Sequence Planning

Plan the breadth (scope) and order (sequence) of curriculum content.

### Scope Planning

Define what's included and what's out:

```markdown
## Scope Definition

### Included Topics
- [ ] Topic A: [Brief description, why included]
- [ ] Topic B: [Brief description, why included]
- [ ] Topic C: [Brief description, why included]

### Excluded Topics (and why)
- [ ] Topic X: [Reason for exclusion, e.g., covered elsewhere, too advanced]
- [ ] Topic Y: [Reason for exclusion]

### Depth Guidelines
| Topic | Depth | Rationale |
|-------|-------|-----------|
| Core concepts | Deep (3+ hours) | Essential for certification |
| Supporting skills | Moderate (1-2 hours) | Needed for practice |
| Context/background | Light (overview only) | Nice to have |
```

### Sequencing Strategies

| Strategy | Description | Best For |
|----------|-------------|----------|
| **Simple to Complex** | Start easy, build difficulty | Skill-building courses |
| **Known to Unknown** | Connect to prior knowledge | Adult learners |
| **Chronological** | Follow time sequence | History, processes |
| **Thematic** | Group by topic/theme | Broad survey courses |
| **Problem-Centered** | Organize around problems | Professional training |
| **Spiral** | Revisit topics at deeper levels | Long-term programs |

### Sequence Template

```markdown
## Course Sequence Plan

### Prerequisites (Before Course)
- Prerequisite 1
- Prerequisite 2

### Module Sequence

| Order | Module | Prerequisites | Key Outcomes |
|-------|--------|---------------|--------------|
| 1 | Foundations | None | LO1, LO2 |
| 2 | Core Skills | Module 1 | LO3, LO4 |
| 3 | Application | Modules 1-2 | LO5 |
| 4 | Advanced | Modules 1-3 | LO6, LO7 |
| 5 | Capstone | All | All LOs integrated |

### Time Allocation

| Module | Hours | % of Total |
|--------|-------|------------|
| Module 1 | 3 | 20% |
| Module 2 | 4 | 27% |
| Module 3 | 3 | 20% |
| Module 4 | 3 | 20% |
| Module 5 | 2 | 13% |
| **Total** | **15** | **100%** |
```

## Evaluation Models

### Kirkpatrick's Four Levels

The standard model for training evaluation:

```
LEVEL 4: RESULTS (Business Impact)
├── What organizational outcomes improved?
├── ROI, KPIs, business metrics
└── Collected: 6-12 months after training
         ↑
LEVEL 3: BEHAVIOR (Transfer)
├── Are learners applying skills on the job?
├── Observable behavior change
└── Collected: 1-3 months after training
         ↑
LEVEL 2: LEARNING (Mastery)
├── Did learners acquire knowledge/skills?
├── Quiz scores, practical assessments
└── Collected: During/immediately after training
         ↑
LEVEL 1: REACTION (Satisfaction)
├── Did learners enjoy the training?
├── Satisfaction surveys, feedback
└── Collected: Immediately after training
```

### Evaluation Methods by Level

| Level | Methods | Timing | Who Collects |
|-------|---------|--------|--------------|
| **1 - Reaction** | Surveys, feedback forms, Net Promoter Score | End of session/course | Facilitator/LMS |
| **2 - Learning** | Quizzes, skills tests, practical demos | During/end of training | LMS/Assessor |
| **3 - Behavior** | Manager observation, 360 feedback, work samples | 30-90 days post | Supervisor |
| **4 - Results** | KPIs, performance metrics, ROI analysis | 6-12 months post | Organization |

### Evaluation Planning Template

```markdown
## Evaluation Plan: [Course/Workshop Name]

### Level 1: Reaction
**Question**: Did participants find the training valuable and engaging?
**Method**: End-of-session survey
**Target**: 4.0/5.0 average satisfaction
**When**: Immediately after training
**Who**: Facilitator distributes, participants complete

### Level 2: Learning
**Question**: Did participants achieve the learning objectives?
**Method**:
- Quizzes after each module (formative)
- Final assessment (summative)
- Practical demonstration
**Target**: 80% achieve passing score (70%)
**When**: During and at end of training
**Who**: LMS auto-scores, facilitator observes demos

### Level 3: Behavior
**Question**: Are participants applying what they learned?
**Method**:
- 30-day follow-up survey
- Supervisor observation checklist
- Work sample review
**Target**: 60% demonstrate behavior change
**When**: 30-60 days post-training
**Who**: Supervisor completes checklist

### Level 4: Results
**Question**: Did training improve organizational outcomes?
**Method**:
- Compare KPIs pre/post
- Measure specific metrics: [list metrics]
**Target**: [Specific improvement targets]
**When**: 6-12 months post-training
**Who**: Management reviews metrics
```

## Templates & Checklists

### Course Design Checklist

```markdown
## Course Design Checklist

### Analysis Phase
- [ ] Target audience defined
- [ ] Learning needs assessed (TNA completed)
- [ ] Prerequisites identified
- [ ] Constraints documented (time, tech, language)
- [ ] Success metrics defined

### Design Phase
- [ ] Learning objectives written (Bloom's verbs)
- [ ] Objectives aligned to business needs
- [ ] Module structure planned (3-7 modules)
- [ ] Lesson sequence logical
- [ ] Assessment strategy aligned to objectives
- [ ] Curriculum map created

### Development Phase
- [ ] Content created for all lessons
- [ ] Videos scripted and recorded (max 15 min each, optimal 8-12 min)
- [ ] Text content written (Grade 8 reading level)
- [ ] Quizzes created (5-10 questions each)
- [ ] Feedback messages written
- [ ] Accessibility checked (captions, alt text)

### Implementation Phase
- [ ] Course published to LMS
- [ ] Enrollment settings configured
- [ ] Pricing/access set
- [ ] Welcome message prepared
- [ ] Support resources ready

### Evaluation Phase
- [ ] Evaluation plan in place (Kirkpatrick levels)
- [ ] Surveys created
- [ ] Analytics configured
- [ ] Improvement process defined
```

```markdown
## Workshop Design Checklist

- [ ] Session content created

### Facilitation & Evaluation

- [ ] Follow-up plan in place
```

### Learning Objective Writing Template

```markdown
## Learning Objective Template

**Format**: By the end of this [lesson/module/course], learners will be able to [action verb] + [specific content] + [condition/context].

### Examples by Bloom's Level

**Remember**:
- List the 7 cooperative principles from memory.
- Define the term "patronage refund" in your own words.

**Understand**:
- Explain the difference between cooperative and corporate governance.
- Summarize the key requirements of RA 11524.

**Apply**:
- Calculate patronage refunds using the contribution method.
- Demonstrate proper cash handling procedures.

**Analyze**:
- Compare financial statements from two cooperatives to identify strengths.
- Differentiate between operating and non-operating income.

**Evaluate**:
- Assess a cooperative's governance effectiveness using the FRAMES framework.
- Justify a recommendation for capital allocation.

**Create**:
- Design a marketing plan for the cooperative's new product line.
- Develop a 90-day action plan addressing identified weaknesses.
```

### BARMM Education Challenges (Design Context)

| Indicator | BARMM | National | Gap |
