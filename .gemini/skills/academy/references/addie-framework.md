## Analysis Phase

The foundation of effective course design. Skip this and everything else suffers.

### Learner Analysis

**Questions to Answer**:
- Who are the learners? (Cooperative members, managers, government staff)
- What do they already know? (Prerequisites)
- What do they need to learn? (Learning gaps)
- How do they prefer to learn? (Video, reading, hands-on)
- What constraints exist? (Time, technology, location)

**MoroAcademy Target Audiences**:
| Audience | Characteristics | Preferred Methods |
|----------|-----------------|-------------------|
| Cooperative Members | Varied education, limited tech | Simple videos, text, mobile-friendly |
| Cooperative Managers | Business background, time-constrained | Case studies, practical exercises |
| Social Enterprise Leaders | Mission-driven, growth-focused | Strategy frameworks, peer learning |
| Government Staff | Process-oriented, compliance-focused | Clear procedures, documentation |

### Needs Assessment

**Performance Gap Analysis**:
```
Current State: What learners can do now
Desired State: What learners should be able to do
Gap: The difference = training content focus
```

**Methods**:
- Surveys/questionnaires
- Interviews with stakeholders
- Analysis of common errors/complaints
- Review of compliance failures
- Observation of work processes

### Context Analysis

**Technical Constraints**:
- Internet connectivity (urban vs rural BARMM)
- Device access (mobile-first design for most)
- Time availability (working professionals)

**Organizational Constraints**:
- Budget for training
- Release time for employees
- Management support
- Existing training culture

## Design Phase

Transform analysis findings into a learning blueprint.

### Learning Objectives

**Bloom's Taxonomy Application**:

```
Level 1: Remember (Recall facts)
  Verbs: define, list, name, identify, recall
  Example: "List the 7 cooperative principles"

Level 2: Understand (Comprehend meaning)
  Verbs: explain, describe, summarize, interpret
  Example: "Explain how patronage refunds benefit members"

Level 3: Apply (Use in new situations)
  Verbs: calculate, demonstrate, apply, implement
  Example: "Calculate member equity distribution"

Level 4: Analyze (Break into components)
  Verbs: compare, contrast, analyze, differentiate
  Example: "Compare audit requirements for different coop types"

Level 5: Evaluate (Make judgments)
  Verbs: assess, justify, evaluate, recommend
  Example: "Evaluate a cooperative's financial health using PEARLS"

Level 6: Create (Produce new work)
  Verbs: design, create, develop, formulate
  Example: "Design a business plan for a new cooperative venture"
```

**Objective Writing Formula**:
```
ABCD Method:
A - Audience: Who are the learners?
B - Behavior: What will they do? (observable verb)
C - Condition: Under what conditions?
D - Degree: To what standard?

Example:
"Given a sample financial statement (C), cooperative treasurers (A)
will calculate (B) the patronage refund distribution with 90% accuracy (D)."
```

### Content Sequencing

**Sequencing Strategies**:

1. **Simple to Complex**: Start with basics, build to advanced
2. **Known to Unknown**: Connect new content to prior knowledge
3. **Chronological**: Follow natural time sequence
4. **Part to Whole**: Teach components, then integration
5. **Problem-Centered**: Present problem, then solutions

**Module Planning Template**:
```
Module Title: _______________
Duration: ______ hours

Prerequisites: What must learners already know?
Learning Objectives: 3-5 measurable statements
Key Topics: Main content areas
Activities: How learners will engage
Assessment: How mastery will be verified
Resources: Materials needed
```

**Alignment Matrix**:

| List 7 principles | Principles video | Match exercise | Quiz: MC |
| Calculate refund | Calculation lesson | Worksheet | Quiz: Problem |

## Development Phase

Create the actual content and materials.

### Video Production Guidelines

**Pre-Production**:
- Script with timing (150 words = ~1 minute)
- Storyboard visual elements
- Prepare presenter notes
- Test recording setup

**Recording Standards**:
- Resolution: 1080p minimum
- Audio: Clear, minimal background noise
- Lighting: Well-lit face, no harsh shadows
- Framing: Head and shoulders, eyes at upper third

**Video Structure** (8-minute optimal):
```
0:00-0:30 - Hook and objective
0:30-6:00 - Core content (3-4 key points)
6:00-7:00 - Example/demonstration
7:00-7:30 - Summary/key takeaways
7:30-8:00 - Call to action/next steps
```

### Text Content Standards

**Readability Guidelines**:
- Target: 8th-grade reading level
- Short paragraphs (3-4 sentences)
- Bullet points for lists
- Bold for key terms
- Headers every 200-300 words

**Content Elements**:
```markdown
## Topic Title

**Key Point**: One sentence summary

### Explanation
2-3 paragraphs of explanation with examples.

> **Tip**: Practical advice in callout box

### Example
Real-world case from BARMM context

### Summary
- Bullet point recap
- Of main points
- Covered above
```

### Quiz Development

**Question Types and Use Cases**:

| Type | Best For | Example |
|------|----------|---------|
| Multiple Choice | Factual recall, concept recognition | "Which principle emphasizes democratic control?" |
| True/False | Quick fact checks | "Cooperatives can distribute profits to non-members. T/F" |
| Fill-in-Blank | Terminology, formulas | "The formula for patronage refund is ___" |
| Matching | Associations, categories | "Match each officer to their primary duty" |
| Short Answer | Application, explanation | "Explain why cooperatives need reserves" |

**Question Writing Guidelines**:
- One clear question per item
- Plausible distractors (wrong answers)
- Avoid "all of the above" and "none of the above"
- No negative phrasing ("Which is NOT...")
- Randomize answer positions

## Implementation Phase

Deploy the course and support learners.

### Course Launch Checklist

**Technical Readiness**:
- [ ] All lessons published and ordered
- [ ] Quizzes functional with correct answers
- [ ] Certificate generation tested
- [ ] Mobile responsiveness verified
- [ ] Video playback tested on slow connections

**Content Readiness**:
- [ ] Learning objectives visible
- [ ] Prerequisites clearly stated
- [ ] Estimated duration accurate
- [ ] Downloads/resources attached
- [ ] Instructor information complete

### Learner Support

**Communication Plan**:
- Welcome message on enrollment
- Progress reminders (weekly for inactive learners)
- Completion congratulations
- Certificate issuance notification

**Common Issues and Solutions**:
| Issue | Solution |
|-------|----------|
| Can't play video | Check connection, try lower quality |
| Quiz not submitting | Clear cache, try different browser |
| Progress not saving | Ensure logged in, refresh page |
| Certificate not appearing | Complete all required lessons |

## Evaluation Phase

Measure effectiveness and improve.

### Kirkpatrick's Four Levels

**Level 1: Reaction**
- Did learners enjoy the course?
- Was content relevant?
- Methods: End-of-course surveys, ratings

**Level 2: Learning**
- Did learners acquire knowledge/skills?
- Can they demonstrate competency?
- Methods: Pre/post tests, assessments

**Level 3: Behavior**
- Are learners applying what they learned?
- Has job performance changed?
- Methods: Manager feedback, observation, follow-up surveys

**Level 4: Results**
- Did training impact organizational goals?
- Measurable business outcomes?
- Methods: Performance metrics, compliance rates, financial indicators

### Evaluation Instruments

**Course Feedback Survey**:
```
1. The course objectives were clear. (1-5)
2. The content was relevant to my work. (1-5)
3. The course length was appropriate. (1-5)
4. The videos/materials were high quality. (1-5)
5. I can apply what I learned. (1-5)
6. I would recommend this course. (1-5)
7. What did you find most valuable? [Open]
8. What should be improved? [Open]
```

**Knowledge Assessment**:
- Pre-test: Baseline knowledge before course
- Post-test: Knowledge after completion
- Learning gain = Post score - Pre score

### Continuous Improvement

**Data to Monitor**:
- Enrollment numbers
- Completion rates
- Average quiz scores
- Time spent per lesson
- Drop-off points
- Learner feedback scores

**Improvement Triggers**:
| Indicator | Threshold | Action |
|-----------|-----------|--------|
| Completion rate | <60% | Review course length, content |
| Quiz pass rate | <70% | Review question difficulty, content |
| Avg rating | <3.5/5 | Analyze feedback, improve |
| Drop-off spike | >30% at point | Redesign that lesson |

## Iterative Design

ADDIE is not strictly linear. Expect to:
- Return to Analysis when evaluation reveals gaps
- Redesign based on implementation feedback
- Develop new content as needs emerge
- Re-implement updated versions

**Version Control**:
- Document all changes
- Track what works and what doesn't
- Build on successful patterns
- Archive lessons learned
