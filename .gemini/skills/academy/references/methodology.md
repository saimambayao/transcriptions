# MoroAcademy Instructional Design Methodology

A research-based methodology for designing engaging online courses and live workshops that maximize learning outcomes while preventing learner fatigue.

> **Core Philosophy**: Learning should be an experience, not an endurance test.

## The Engagement Problem

### Why Learners Disengage

| Factor | Online Courses | Live Workshops |
|--------|---------------|----------------|
| **Cognitive overload** | Too much content per video | Lecture-heavy sessions |
| **Passive consumption** | Watch without doing | Listen without participating |
| **Irrelevance** | Generic examples | Disconnected from work reality |
| **Monotony** | Same format throughout | Predictable structure |
| **Isolation** | No peer interaction | No collaborative activities |
| **No stakes** | No immediate application | No tangible outputs |

### The Cost of Boredom

- **Online**: 50-70% dropout rates, skimming without learning
- **Workshops**: Mental checkout, phone scrolling, minimal retention
- **Both**: Certificates without competence, wasted ETF funds

## Foundational Principles

### 1. Andragogy (Adult Learning Theory)

Adults learn differently than children. Apply Knowles' six principles:

| Principle | Application |
|-----------|-------------|
| **Self-Concept** | Give learners control over pace and path |
| **Experience** | Build on what they already know from coop work |
| **Readiness** | Connect learning to their current role challenges |
| **Orientation** | Focus on problem-solving, not abstract theory |
| **Motivation** | Emphasize practical benefits and career growth |
| **Need to Know** | Explain WHY before WHAT |

### 2. Cognitive Load Theory

The brain has limited working memory. Manage it carefully:

| Overload Type | Problem | Solution |
|---------------|---------|----------|
| **Intrinsic** | Topic is complex | Break into smaller chunks |
| **Extraneous** | Poor design adds confusion | Simplify visuals, clear navigation |
| **Germane** | Good effort for learning | Add worked examples, practice |

**Practical Rule**: One concept per learning segment. If you can't explain it in 3 minutes, break it down further.

### 3. Spaced Repetition

Learning strengthens through repeated exposure over time:

```
Day 1: Learn concept
Day 2: Quick review (quiz, flashcard)
Day 7: Apply in context
Day 30: Integrate with new learning
```

**Application**: Design courses with built-in review moments. Reference earlier concepts in later modules.

### 4. Active Learning Superiority

Research consistently shows active learning outperforms passive:

| Method | Average Retention |
|--------|-------------------|
| Lecture | 5% |
| Reading | 10% |
| Audiovisual | 20% |
| Demonstration | 30% |
| Discussion | 50% |
| Practice by Doing | 75% |
| Teaching Others | 90% |

**Implication**: Minimize lecture, maximize doing.

### 5. Flow State Design

Csikszentmihalyi's Flow: optimal engagement when challenge matches skill.

```
                    ┌─────────────────────────────────┐
                    │           ANXIETY               │
        Challenge   │    (too hard, frustrating)      │
            ▲       ├─────────────────────────────────┤
            │       │           FLOW                  │
            │       │    (optimal engagement)         │
            │       ├─────────────────────────────────┤
            │       │          BOREDOM                │
            │       │    (too easy, disengaging)      │
            └───────┴─────────────────────────────────┘
                              Skill Level →
```

**Application**: Start easy, progressively increase difficulty. Provide scaffolding for complex tasks.

## Online Course Methodology

### The Microlearning Architecture

Break content into digestible segments that respect attention spans.

#### Video Duration Guidelines

| Duration | Purpose | Content Type |
|----------|---------|--------------|
| **3 min** | Quick concept | Single definition, one idea |
| **5 min** | Concept + example | Idea with illustration |
| **7 min** | Concept + multiple examples | Idea with 2-3 applications |
| **10 min** | Process walkthrough | Step-by-step procedure |
| **12 min** | Complex topic | Multi-part concept with practice prompt |
| **15 min** | Maximum | Comprehensive coverage (rare) |

**Research Basis**: MIT/edX studies show engagement drops significantly after 6 minutes. Optimal is 6 minutes or less.

#### The 3-3-3 Video Structure

Every video follows this rhythm:

```
┌─────────────────────────────────────────────────────────────┐
│  HOOK (30 sec)                                              │
│  "Why should I care about this?"                            │
│  - Start with a problem, question, or surprising fact       │
│  - Connect to learner's real challenges                     │
├─────────────────────────────────────────────────────────────┤
│  CONTENT (Main Duration)                                    │
│  Chunked into 3 segments max:                               │
│  1. Explain the concept                                     │
│  2. Show how it works (example/demo)                        │
│  3. Connect to their context                                │
├─────────────────────────────────────────────────────────────┤
│  BRIDGE (30 sec)                                            │
│  "What do I do with this?"                                  │
│  - Key takeaway (one sentence)                              │
│  - Transition to next lesson or activity                    │
│  - Reflection prompt or mini-challenge                      │
└─────────────────────────────────────────────────────────────┘
```

#### Lesson Variety Pattern

Never use the same format twice in a row:

```
Module Structure (Example: 1.5 hours)
│
├── Lesson 1: VIDEO (7 min) - Introduce concept
├── Lesson 2: TEXT (10 min) - Deep dive with examples
├── Lesson 3: VIDEO (5 min) - Demonstration
├── Lesson 4: QUIZ (5 min) - Check understanding
├── Lesson 5: VIDEO (7 min) - Advanced application
├── Lesson 6: TEXT (8 min) - Case study analysis
├── Lesson 7: ASSIGNMENT (15 min) - Apply to own coop
└── Lesson 8: QUIZ (5 min) - Module assessment
```

#### Interactive Elements

Transform passive watching into active learning:

| Element | Purpose | Frequency |
|---------|---------|-----------|
| **Reflection Prompts** | Pause to think before answer revealed | Every video |
| **Knowledge Checks** | 1-2 questions mid-video | Videos > 7 min |
| **Scenario Choices** | "What would you do?" branches | Per module |
| **Downloadable Templates** | Tools they'll actually use | Per unit |
| **Discussion Forums** | Peer learning, questions | Per course |

#### The "Watch-Pause-Try" Method

Build pauses into the learning flow:

```
VIDEO: "Here's how to calculate patronage refund..."
       [Demonstration with example]

       "Now pause this video and try calculating
        for your own cooperative using the template
        you downloaded. Come back when you're done."

       [5-second pause screen with timer suggestion]

       "Welcome back. Let's check your work..."
       [Review common mistakes, correct approach]
```

### Online Course Engagement Techniques

#### 1. Story-Driven Content

Frame every module around a narrative:

```markdown
## Module: Financial Statement Analysis

**The Story**: Meet Cooperative Mindanao Farmers (CMF).
They've been growing for 3 years but the treasurer just
resigned. New treasurer Maria needs to understand the
financials quickly. Follow her journey as she learns
to read and analyze the cooperative's financial health.

Lesson 1: Maria's First Day (The Balance Sheet)
Lesson 2: Where Did the Money Go? (Income Statement)
Lesson 3: Maria Spots a Problem (Ratio Analysis)
Lesson 4: Presenting to the BOD (Financial Reporting)
```

#### 2. Progressive Disclosure

Don't overwhelm. Reveal complexity gradually:

```
Level 1: "A balance sheet shows what the coop owns and owes"
         [Simple visual, two columns]

Level 2: "Assets are categorized by how quickly
          they can become cash"
         [Add current vs non-current breakdown]

Level 3: "Now let's look at how these connect to
          the income statement"
         [Add relationships and ratios]
```

#### 3. Gamification Elements

Add game mechanics without being childish:

| Element | Implementation |
|---------|----------------|
| **Progress bars** | Visual completion percentage |
| **Badges** | "Completed Unit 1", "Quiz Master", "First Certificate" |
| **Streaks** | "5-day learning streak" |
| **Leaderboards** | Optional, per cohort/coop |
| **Unlockables** | Advanced content after prerequisites |

#### 4. Social Learning

Combat isolation in self-paced learning:

- **Cohort mode**: Groups start together, discussion deadlines
- **Peer review**: Assignments reviewed by fellow learners
- **Study groups**: Suggested groupings for cooperative members
- **Live Q&A sessions**: Monthly with course facilitator

## Live Workshop Methodology

### The 30/70 Rule

Maximum 30% presentation, minimum 70% participation.

```
Traditional (Boring):        Engaging:
┌─────────────────────┐      ┌─────────────────────┐
│ ████████████████ 80%│      │ ██████ 30%         │ ← Lecture
│ ████ 20%            │      │ ██████████████ 70% │ ← Activity
└─────────────────────┘      └─────────────────────┘
     Lecture-heavy               Activity-rich
```

### Workshop Session Types

Design sessions using these activity types:

| Type | Duration | Energy Level | Purpose |
|------|----------|--------------|---------|
| **Lecture** | 15-20 min max | Low | Introduce concepts |
| **Discussion** | 15-30 min | Medium | Process ideas |
| **Workshop** | 45-90 min | High | Create outputs |
| **Simulation** | 60-120 min | Very High | Practice decisions |
| **Gallery Walk** | 20-30 min | Medium | Share and learn |
| **Reflection** | 10-15 min | Low | Consolidate learning |

### The 90-Minute Attention Cycle

Human attention follows natural cycles. Design around them:

```
┌─────────────────────────────────────────────────────────────────┐
│                        90-MINUTE BLOCK                          │
├──────────────────┬──────────────────┬──────────────────────────┤
│    Opening       │      Core        │        Closing           │
│    (15 min)      │    (60 min)      │       (15 min)           │
├──────────────────┼──────────────────┼──────────────────────────┤
│ • Energizer      │ • Content chunk 1│ • Key takeaways          │
│ • Review prior   │   (20 min)       │ • Preview next           │
│ • Set objectives │ • Activity 1     │ • Reflection prompt      │
│                  │   (20 min)       │                          │
│                  │ • Content chunk 2│                          │
│                  │   (10 min)       │                          │
│                  │ • Activity 2     │                          │
│                  │   (10 min)       │                          │
└──────────────────┴──────────────────┴──────────────────────────┘
                              │
                              ▼
                         BREAK (15-20 min)
                              │
                              ▼
                    [Next 90-minute block]
```

### Daily Energy Arc

Plan session intensity around natural energy patterns:

```
Energy Level
     ▲
     │        ┌───┐
High │    ┌───┤   ├───┐              ┌───┐
     │    │   │   │   │          ┌───┤   │
Med  │────┤   │   │   ├──────────┤   │   ├────
     │    │   │   │   │          │   │   │
Low  │    │   │   │   │    ┌─────┤   │   │
     └────┴───┴───┴───┴────┴─────┴───┴───┴────→ Time
          8am  10am  12pm  1pm    3pm   5pm

          ↑        ↑       ↑           ↑
       Warm-up  Peak    Post-lunch  Afternoon
                        slump       recovery
```

**Session Planning by Time**:

| Time | Energy | Best Activities |
|------|--------|-----------------|
| 8:00-9:00 | Rising | Opening, team building, overview |
| 9:00-10:30 | Peak | Complex content, challenging activities |
| 10:30-10:45 | — | **BREAK** |
| 10:45-12:00 | High | Workshops, hands-on practice |
| 12:00-1:00 | — | **LUNCH** |
| 1:00-2:00 | Low | Light activity, group discussion, energizers |
| 2:00-3:30 | Recovering | Practical work, simulations |
| 3:30-3:45 | — | **BREAK** |
| 3:45-5:00 | Moderate | Synthesis, action planning, closing |

### Energizers and State Changes

Combat post-lunch slump and attention fatigue:

| Type | Duration | When to Use |
|------|----------|-------------|
| **Physical energizer** | 3-5 min | After lunch, mid-afternoon |
| **Brain teaser** | 5-7 min | Between heavy content |
| **Quick share** | 5-10 min | Start of new session |
| **Movement activity** | 10-15 min | Every 90 minutes |
| **Music break** | 3-5 min | Transitions |

**Examples**:
- "Stand up and find someone from a different coop to discuss..."
- "Everyone move to a different table for the next activity"
- "Quick brainstorm: Post-it notes, 1 idea per note, 2 minutes"
- "Gallery walk: View all team outputs, leave feedback"

### Workshop Output Design

Every workshop should produce tangible outputs:

| Output Type | Purpose | Example |
|-------------|---------|---------|
| **Template filled** | Immediate application | Completed strategic plan template |
| **Action plan** | Next steps commitment | 90-day implementation plan |
| **Draft document** | Work product started | Draft credit policy |
| **Peer feedback** | Social accountability | Reviewed business plan |
| **Presentation** | Teach to learn | Group presentation on case study |

**The Output Promise**: "By the end of this workshop, you will have [tangible thing] you can use immediately in your cooperative."

### Kolb's Experiential Learning Cycle

Structure workshop sessions around the complete learning cycle:

```
        ┌─────────────────────────────────────┐
        │                                     │
        ▼                                     │
   ┌─────────┐                          ┌─────────┐
   │CONCRETE │                          │ ACTIVE  │
   │EXPERIENCE│ ◄──────────────────────►│EXPERIMENT│
   │ (Doing) │                          │(Trying) │
   └────┬────┘                          └────▲────┘
        │                                    │
        │       The Learning Cycle           │
        │                                    │
        ▼                                    │
   ┌─────────┐                          ┌─────────┐
   │REFLECTIVE│                          │ABSTRACT │
   │OBSERVATION│ ◄──────────────────────►│CONCEPTUAL│
   │(Watching)│                          │(Thinking)│
   └─────────┘                          └─────────┘
```

**Session Design Pattern**:

1. **Experience** (20 min): Simulation, case study, or activity
2. **Reflect** (15 min): What happened? What did you notice?
3. **Conceptualize** (15 min): What principles emerge? Theory input
4. **Apply** (20 min): How will you use this? Plan next steps

## The ENGAGE Framework

A unified methodology for both online and workshop design.

### E - Experience First

Start with experience, not explanation.

| Format | Experience-First Approach |
|--------|---------------------------|
| **Online** | Open with scenario/problem before teaching concept |
| **Workshop** | Begin with simulation or case before lecture |

```
❌ Traditional: "Today we'll learn about financial ratios..."
✅ Experience-first: "Your coop's liquidity ratio is 0.8.
                      Members are asking for withdrawals.
                      What do you do?"
```

### N - Narrative Thread

Weave a story throughout the learning journey.

| Format | Narrative Approach |
|--------|-------------------|
| **Online** | Recurring character/coop case across modules |
| **Workshop** | Multi-day simulation with evolving scenario |

```
Module 1: CMF Cooperative is struggling with member retention
Module 2: They discover financial management issues
Module 3: New treasurer implements changes
Module 4: Six months later - measuring the results
```

### G - Guided Practice

Provide scaffolded practice with feedback.

| Format | Guided Practice |
|--------|-----------------|
| **Online** | Worked examples → Practice problems → Self-check |
| **Workshop** | Demo → Pair practice → Solo application → Peer review |

```
Step 1: Watch me calculate the debt-to-equity ratio
Step 2: Let's do one together (I guide, you follow)
Step 3: Try one yourself (I'll check your answer)
Step 4: Now do your cooperative's actual ratio
```

### A - Active Processing

Require active engagement, not passive consumption.

| Format | Active Elements |
|--------|-----------------|
| **Online** | Pause prompts, quizzes, reflection journals, assignments |
| **Workshop** | Discussions, activities, workshops, simulations |

**Minimum Activity Ratio**:
- Online: Activity every 5-7 minutes of video
- Workshop: Activity every 15-20 minutes of lecture

### G - Group Learning

Leverage social learning power.

| Format | Group Elements |
|--------|----------------|
| **Online** | Discussion forums, peer review, cohort learning |
| **Workshop** | Table groups, pair work, team challenges |

**Group Size Guidelines**:
- Pairs: Quick discussions, peer review
- Small groups (3-5): Problem-solving, case analysis
- Large groups (6-10): Diverse perspectives, simulations
- Whole room: Sharing, synthesis, celebration

### E - Extend Beyond

Connect learning to real application.

| Format | Extension Strategies |
|--------|---------------------|
| **Online** | Action plans, workplace assignments, follow-up modules |
| **Workshop** | 30/60/90-day plans, accountability partners, reunion sessions |

```
"Before you leave today, write down:
 1. One thing you'll do THIS WEEK
 2. One thing you'll do THIS MONTH
 3. One person who will hold you accountable"
```

## Content Design Patterns

### Pattern 1: Problem-Centered Design

Structure content around problems learners face:

```
Module: Managing Delinquent Loans

Problem 1: How do I identify at-risk loans early?
  → Early warning indicators
  → Monitoring systems

Problem 2: How do I approach delinquent members?
  → Collection strategies
  → Communication scripts

Problem 3: When should I escalate to legal action?
  → Decision framework
  → Cost-benefit analysis
```

### Pattern 2: Case-Based Learning

Build modules around realistic cooperative cases:

```markdown
## Case: Barangay Fishermen's Cooperative

**Background**: 85 members, ₱2.3M assets, 3 years old
**Situation**: Members want to expand into ice plant,
              but liquidity is low and loans are increasing
**Data**: [Provide financial statements, member survey]

**Tasks**:
1. Analyze their financial position
2. Evaluate the expansion proposal
3. Present recommendation to the BOD
```

### Pattern 3: Skill Ladder Design

Build competence progressively:

```
Level 1: Awareness
  "What is a balance sheet?"

Level 2: Understanding
  "How do assets and liabilities relate?"

Level 3: Application
  "Analyze this cooperative's balance sheet"

Level 4: Analysis
  "What does this trend tell us?"

Level 5: Evaluation
  "Is this cooperative financially healthy?"

Level 6: Creation
  "Design a financial monitoring system"
```

### Pattern 4: Worked Example Pairs

For procedural skills, use worked example → practice pairs:

```
WORKED EXAMPLE:
"Here's how to calculate the debt-to-equity ratio:

 Debt-to-Equity = Total Liabilities / Total Equity

 For CMF Cooperative:
 Total Liabilities = ₱500,000
 Total Equity = ₱1,000,000

 Debt-to-Equity = ₱500,000 / ₱1,000,000 = 0.5

 This means for every ₱1 of equity, they have ₱0.50 in debt.
 This is a healthy ratio (below 1.0)."

YOUR TURN:
"Now calculate for your cooperative:
 Total Liabilities = ₱_______
 Total Equity = ₱_______

 Debt-to-Equity = ______ / ______ = ______

 Is this healthy? Why or why not?"
```

## Energy and Attention Management

### The Attention Span Reality

| Format | Sustained Attention | Implication |
|--------|--------------------| ------------|
| **Video** | 6-10 minutes | Max 15 min videos, ideal 5-7 min |
| **Lecture** | 15-20 minutes | Break every 15 min with activity |
| **Activity** | 45-90 minutes | Varies by engagement level |
| **Full day** | 4-5 productive hours | Plan for natural dips |

### State Change Triggers

When attention drops, change the state:

| State Change | How It Works |
|--------------|--------------|
| **Physical** | Stand up, move, change seats |
| **Social** | Turn to partner, form new groups |
| **Sensory** | Watch video, listen to audio, handle materials |
| **Cognitive** | Switch from receiving to producing |
| **Emotional** | Inject humor, tell story, show impact |

### Online Attention Sustainers

| Technique | Implementation |
|-----------|----------------|
| **Pattern interrupt** | Unexpected visual, sound, or question |
| **Prediction prompt** | "Before I show you, guess what happens..." |
| **Fill-in-blank** | Pause for learner to complete thought |
| **Controversy** | Present opposing viewpoints |
| **Personal story** | Relatable human element |

### Workshop Energy Management

| Time | If Energy Low | Do This |
|------|---------------|---------|
| Morning start | Sleepy arrivals | Physical energizer, music |
| Post-lunch | Food coma | Movement activity, hands-on task |
| Mid-afternoon | Slump | Change rooms, outdoor activity |
| Late afternoon | Fatigue | Collaborative work, clear endpoint |

## Assessment That Motivates

### Assessment Philosophy

Assessment should:
- ✅ Reinforce learning (not just measure it)
- ✅ Feel achievable (not defeating)
- ✅ Provide useful feedback (not just scores)
- ✅ Connect to real work (not just theory)

### Online Course Assessment

| Type | Purpose | Design Tips |
|------|---------|-------------|
| **Lesson quiz** | Check understanding | 2-3 questions, immediate feedback |
| **Module quiz** | Consolidate learning | 5-10 questions, 70% pass, retry allowed |
| **Assignments** | Apply to own context | Template-based, practical output |
| **Final exam** | Certify competence | Comprehensive, case-based scenarios |

**Quiz Question Best Practices**:
- Mix question types (multiple choice, true/false, matching)
- Include scenario-based questions
- Provide explanation for correct AND incorrect answers
- Allow retakes (learning > punishment)

### Workshop Assessment

| Type | Purpose | Design Tips |
|------|---------|-------------|
| **Pre-assessment** | Baseline, tailor content | Simple, non-threatening |
| **Formative checks** | Adjust facilitation | Quick polls, thumbs up/down |
| **Output review** | Apply learning | Template completion, peer feedback |
| **Post-assessment** | Measure growth | Compare to pre, celebrate gains |
| **Observation** | Competence verification | For certification-level skills |

### The 3-2-1 Reflection

End every session with structured reflection:

```
Write down:
3 things you learned today
2 things you'll apply this week
1 question you still have
```

## BARMM Contextualization

### General → BARMM Pattern

All content should follow this structure:

```
1. Universal Concept (What everyone teaches)
   "Net surplus distribution follows statutory requirements..."

2. Philippine Application (RA 9520, CDA)
   "Under RA 9520 Article 86, cooperatives must allocate..."

3. BARMM Contextualization (CSEA, local context)
   "In BARMM, CSEA provides additional guidance for..."
   "For Bangsamoro cooperatives, consider..."
```

### Language Considerations

For BARMM's ESL beginner audience:

| Aspect | Guideline |
|--------|-----------|
| **Pace** | 90 words per minute (slower than standard) |
| **Vocabulary** | Define technical terms in Filipino/local language |
| **Visuals** | Heavy use of diagrams, less text |
| **Examples** | Local cooperative names, BARMM scenarios |
| **Script pattern** | "Ang [English term], o [Filipino explanation]..." |

### Cultural Sensitivity

| Element | Application |
|---------|-------------|
| **Islamic values** | Align examples with halal practices |
| **Community orientation** | Emphasize bayanihan, collective benefit |
| **Respect hierarchy** | Acknowledge elders, leaders appropriately |
| **Local examples** | Use BARMM cooperatives, local products |
| **Visual representation** | Include diverse Bangsamoro imagery |

### Local Case Studies

Build a library of BARMM-specific cases:

```
- Marawi rehabilitation cooperative challenges
- Coastal fishing cooperative success stories
- Agricultural cooperative market linkage examples
- Women's cooperative empowerment cases
- Youth cooperative innovation stories
```

## Methodology Checklist

### Before Designing

- [ ] Defined clear learning objectives (Bloom's verbs)
- [ ] Identified learner profile (role, experience, constraints)
- [ ] Determined delivery format (online, workshop, blended)
- [ ] Set measurable success criteria

### During Design

**For Online Courses**:
- [ ] Videos under 15 minutes (ideal: 5-7 min)
- [ ] Variety of lesson types (video, text, quiz)
- [ ] Activity every 5-7 minutes of content
- [ ] Reflection prompts and practice opportunities
- [ ] Downloadable tools/templates
- [ ] Story/narrative thread throughout

**For Live Workshops**:
- [ ] 30/70 rule (max 30% lecture)
- [ ] 90-minute attention blocks with breaks
- [ ] Energy management across the day
- [ ] Tangible outputs for each session
- [ ] Group activities and discussions
- [ ] State changes every 15-20 minutes

### After Design

- [ ] Peer review by subject matter expert
- [ ] Pilot test with sample learners
- [ ] Iterate based on feedback
- [ ] Document facilitator notes (for workshops)
- [ ] Create assessment rubrics
