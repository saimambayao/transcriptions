## Lean Startup Methodology

### Eric Ries (2011)

**Core Premise**: Startups can shorten their product development cycles by adopting a combination of business-hypothesis-driven experimentation, iterative product releases, and validated learning.

**Five Principles:**

| Principle | Description |
|-----------|-------------|
| **Entrepreneurs are everywhere** | Startups can exist in any size organization |
| **Entrepreneurship is management** | Startups need management suited to uncertainty |
| **Validated learning** | Learn through scientific experimentation |
| **Build-Measure-Learn** | Accelerate through the feedback loop |
| **Innovation accounting** | Measure progress with actionable metrics |

**Build-Measure-Learn Loop:**

```text
┌─────────────────────────────────────────────────┐
│                                                 │
│    IDEAS                                        │
│      │                                          │
│      │ Build (minimize time)                    │
│      ▼                                          │
│    PRODUCT ───────────────────►  CODE           │
│      │                          MVP             │
│      │ Measure (validated learning)             │
│      ▼                                          │
│    DATA ─────────────────────►  METRICS         │
│      │                         Analytics        │
│      │ Learn (pivot or persevere)               │
│      ▼                                          │
│    INSIGHTS ──────────────────► PIVOT or        │
│      │                          PERSEVERE       │
│      │                                          │
│      └──────────── back to IDEAS ───────────────┘
│                                                 │
└─────────────────────────────────────────────────┘
```

**MVP Types:**
- Concierge MVP: Manual service before automation
- Wizard of Oz MVP: Appears automated, human behind
- Landing Page MVP: Test demand before building
- Video MVP: Demonstrate concept (e.g., Dropbox)
- Piecemeal MVP: Combine existing tools

**Pivot Types:**
- Zoom-in pivot: One feature becomes the whole product
- Zoom-out pivot: Product becomes one feature of larger product
- Customer segment pivot: Same product, different customer
- Customer need pivot: Same customer, different problem
- Platform pivot: Application to platform (or vice versa)
- Business architecture pivot: High margin/low volume ↔ low margin/high volume
- Value capture pivot: Change monetization
- Engine of growth pivot: Viral ↔ sticky ↔ paid
- Channel pivot: Different distribution
- Technology pivot: Different technology, same solution

## Customer Development

### Steve Blank (2005)

**Core Premise**: Startups are not smaller versions of large companies. They are temporary organizations searching for a repeatable and scalable business model.

**Four Steps:**

```text
┌─────────────────────────────────────────────────────────────────┐
│                    CUSTOMER DEVELOPMENT                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  SEARCH                              EXECUTE                     │
│  ──────                              ───────                     │
│                                                                  │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────┐│
│  │   CUSTOMER   │  │   CUSTOMER   │  │   CUSTOMER   │  │COMPANY│
│  │  DISCOVERY   │→ │  VALIDATION  │→ │   CREATION   │→ │BUILDING│
│  └──────────────┘  └──────────────┘  └──────────────┘  └──────┘│
│        │                 │                 │              │      │
│        ▼                 ▼                 ▼              ▼      │
│   Validate          Validate          Scale          Transition │
│   Problem &         Sales &           Acquisition     to        │
│   Solution          Business                          Execution │
│   Hypotheses        Model                                       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**Customer Discovery Process:**

1. **State Hypotheses**: Document assumptions about:
   - Customer problem
   - Product features
   - Business model
   - Pricing

2. **Test Problem Hypotheses**:
   - Conduct problem interviews (15-20 minimum)
   - Validate problem exists and is painful
   - Understand current alternatives

3. **Test Product Hypotheses**:
   - Conduct solution interviews
   - Show mockups/prototypes
   - Validate solution resonates

4. **Verify or Pivot**:
   - Enough evidence? → Move to Customer Validation
   - Not enough? → Pivot and restart

**Interview Techniques:**

| Type | Purpose | Questions |
|------|---------|-----------|
| **Problem Interview** | Validate problem | "Tell me about the last time you experienced X" |
| **Solution Interview** | Validate solution | "If you had Y, how would that help?" |
| **Pricing Interview** | Validate willingness to pay | "What would you pay for this?" |

## Running Lean

### Ash Maurya (2012)

**Core Premise**: A systematic process for iterating from Plan A to a plan that works, before running out of resources.

**Lean Canvas:**

| | Problem | Solution | Unique Value Proposition | Unfair Advantage | Customer Segments |
|-|---------|----------|--------------------------|------------------|-------------------|
| | Top 3 problems | Top 3 features | Single, clear message | Can't be copied/bought | Target customers |
| | Existing alternatives | | High-level concept | | Early adopters |

| Key Metrics | | | | Channels |
|-------------|--|--|--|----------|
| Key activities to measure | | | | Path to customers |

| Cost Structure | | | | Revenue Streams |
|----------------|--|--|--|-----------------|
| Customer acquisition, hosting, people | | | | Revenue model, LTV, pricing |

**Problem-Solution Fit → Product-Market Fit:**

```text
STAGE 1: Problem-Solution Fit          STAGE 2: Product-Market Fit
┌──────────────────────────────┐       ┌──────────────────────────────┐
│ Do I have a problem worth    │       │ Have I built something       │
│ solving?                     │  →    │ people want?                 │
├──────────────────────────────┤       ├──────────────────────────────┤
│ • Problem interviews         │       │ • MVP with paying customers  │
│ • Solution interviews        │       │ • Retention metrics          │
│ • Lean Canvas v1             │       │ • Referral metrics           │
└──────────────────────────────┘       └──────────────────────────────┘
```

## Zero to One

### Peter Thiel (2014)

**Core Premise**: The next Bill Gates will not build an operating system. The next Larry Page will not make a search engine. Copying others takes the world from 1 to n; creating something new goes from 0 to 1.

**Key Concepts:**

| Concept | Description |
|---------|-------------|
| **Definite Optimism** | The future is knowable and better; plan for it |
| **Secrets** | Find truths few people agree with you on |
| **Monopoly** | Aim to dominate a small market, then expand |
| **Last Mover Advantage** | Be the last significant player in your market |
| **Power Law** | A few companies/investments drive most returns |

**Seven Questions Every Business Must Answer:**

1. **Engineering**: Can you create breakthrough technology (10x better)?
2. **Timing**: Is now the right time to start?
3. **Monopoly**: Are you starting with a big share of a small market?
4. **People**: Do you have the right team?
5. **Distribution**: Can you deliver your product?
6. **Durability**: Will your market position be defensible in 10-20 years?
7. **Secret**: Have you identified a unique opportunity others don't see?

## Traction Framework

### Gabriel Weinberg & Justin Mares (2015)

**Core Premise**: Traction trumps everything. Most startups fail not because of product but because of distribution.

**19 Traction Channels:**

| # | Channel | Description |
|---|---------|-------------|
| 1 | Viral Marketing | Users bring more users |
| 2 | PR | Traditional media coverage |
| 3 | Unconventional PR | Publicity stunts |
| 4 | SEM | Paid search advertising |
| 5 | Social/Display Ads | Facebook, Instagram, etc. |
| 6 | Offline Ads | TV, radio, billboards |
| 7 | SEO | Organic search ranking |
| 8 | Content Marketing | Blog, videos, resources |
| 9 | Email Marketing | Newsletters, sequences |
| 10 | Engineering as Marketing | Free tools, calculators |
| 11 | Targeting Blogs | Sponsor or write for blogs |
| 12 | Business Development | Strategic partnerships |
| 13 | Sales | Direct outbound sales |
| 14 | Affiliate Programs | Revenue share with promoters |
| 15 | Existing Platforms | App stores, marketplaces |
| 16 | Trade Shows | Industry events |
| 17 | Offline Events | Meetups, conferences |
| 18 | Speaking Engagements | Thought leadership |
| 19 | Community Building | Forums, user groups |

**Bullseye Framework:**

```text
┌────────────────────────────────────────┐
│           OUTER RING                    │
│    What's possible? (all 19 channels)  │
│                                         │
│     ┌────────────────────────────┐     │
│     │      MIDDLE RING           │     │
│     │   What's probable?         │     │
│     │   (top 6 channels)         │     │
│     │                            │     │
│     │   ┌──────────────────┐    │     │
│     │   │   BULLSEYE        │    │     │
│     │   │  Test top 3       │    │     │
│     │   │  Focus on #1      │    │     │
```

## Application Guidelines

### When to Apply Each Framework

| Situation | Recommended Framework |
|-----------|----------------------|
| Just starting, have an idea | Customer Development |
| Need to validate problem | Lean Startup (Problem Interviews) |
| Building first product | Lean Startup (MVP) |
| Product built, no traction | Traction (Bullseye) |
| Need to pivot | Lean Startup (Pivot Types) |
| Facing hard decisions | Hard Things (Wartime CEO) |
| Building something unique | Zero to One (7 Questions) |
| Planning business model | Running Lean (Lean Canvas) |

### Framework Combinations

```text
TYPICAL STARTUP JOURNEY:
│
├── Month 0-3: Customer Development + Lean Canvas
│   └── Validate problem, draft business model
│
├── Month 3-6: Lean Startup + MVP
│   └── Build, measure, learn
│
├── Month 6-12: Traction + Iteration
│   └── Find scalable channels
│
├── Month 12-24: Zero to One + Hard Things
│   └── Build defensible position
│
└── Month 24+: Transition to /enterprise-dev
    └── Scale operations
```
