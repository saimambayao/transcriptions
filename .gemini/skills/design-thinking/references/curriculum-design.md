## Learning Levels

### Level 1: Foundational (Awareness)

**Target Audience**: First-time learners, general staff, community members

**Duration**: 4-8 hours (1 day workshop or online course)

**Outcomes**:
- Understand what design thinking is
- Know the 5 phases and their purpose
- Experience a mini design challenge
- Apply basic empathy and ideation tools

### Level 2: Practitioner (Application)

**Target Audience**: Project leads, program officers, coop/SE managers

**Duration**: 16-24 hours (2-3 day workshop or multi-week course)

**Outcomes**:
- Facilitate design thinking sessions
- Apply full process to real challenges
- Use diverse tools and methods
- Lead small teams through design sprints

### Level 3: Advanced (Mastery)

**Target Audience**: Trainers, innovation leads, consultants

**Duration**: 40+ hours (week-long intensive or certification program)

**Outcomes**:
- Design and deliver DT training
- Coach teams through complex projects
- Adapt methods for specific contexts
- Build organizational design capability

## Standard Course Modules

### Module 1: Introduction to Design Thinking

**Learning Objectives**:
- Define design thinking and its origins
- Explain why design thinking matters
- Identify where design thinking applies

**Content**:
- What is design thinking? (d.school, IDEO, Double Diamond)
- History and evolution (Herbert Simon to Tim Brown)
- Mindsets of design thinkers
- Case studies (IDEO, public sector, social enterprise)

**Activities**:
- Design thinking timeline exploration
- Mindset self-assessment
- Case study discussion

### Module 2: Empathize - Understanding Users

**Learning Objectives**:
- Conduct effective user interviews
- Apply observation techniques
- Create empathy maps

**Content**:
- Why empathy matters
- Interview techniques (5 Whys, open questions)
- Observation methods (AEIOU framework)
- Empathy mapping (Says, Thinks, Does, Feels)

**Activities**:
- Practice interview in pairs
- Field observation exercise
- Empathy map creation

### Module 3: Define - Framing the Problem

**Learning Objectives**:
- Synthesize research insights
- Create POV statements
- Write effective HMW questions

**Content**:
- From data to insights
- POV statement formula
- How Might We (HMW) questions
- Problem framing principles

**Activities**:
- Insight clustering exercise
- POV statement writing
- HMW brainstorming

### Module 4: Ideate - Generating Solutions

**Learning Objectives**:
- Apply brainstorming rules
- Use structured ideation techniques
- Select ideas for prototyping

**Content**:
- Divergent vs convergent thinking
- Brainstorming rules and techniques
- Crazy 8s, SCAMPER, mind mapping
- Idea selection methods

**Activities**:
- Brainstorming session
- Crazy 8s exercise
- Dot voting for selection

### Module 5: Prototype - Building to Learn

**Learning Objectives**:
- Create rapid prototypes
- Choose appropriate fidelity level
- Use prototypes for learning

**Content**:
- Why prototype? (Build to think)
- Prototype types and fidelity
- Rapid prototyping techniques
- Storyboarding

**Activities**:
- Paper prototyping challenge
- Storyboard creation
- Prototype gallery walk

### Module 6: Test - Learning from Users

**Learning Objectives**:
- Plan and conduct user tests
- Capture feedback effectively
- Iterate based on learning

**Content**:
- Testing principles
- User testing protocol
- Feedback capture methods
- Iteration strategies

**Activities**:
- Test plan development
- Role-play user testing
- Feedback synthesis

### Module 7: Design Thinking in Practice

**Learning Objectives**:
- Apply full process to real challenge
- Work effectively in teams
- Present and pitch solutions

**Content**:
- End-to-end design sprint
- Team collaboration
- Presentation techniques

**Activities**:
- Full design challenge (organization-specific)
- Team presentations
- Peer feedback

### Module 8: Building Design Capability

**Learning Objectives**:
- Plan design thinking initiatives
- Overcome common barriers
- Sustain design practice

**Content**:
- Implementing design thinking
- Common challenges and solutions
- Building design culture
- Next steps and resources

**Activities**:
- Action planning
- Barrier-busting exercise
- Commitment statements

## Workshop Formats

### Format 1: Half-Day Introduction (4 hours)

| Time | Activity |
|------|----------|
| 0:00-0:30 | Welcome, icebreaker |
| 0:30-1:00 | Design thinking overview |
| 1:00-1:30 | Empathy exercise |
| 1:30-2:00 | Define challenge |
| 2:00-2:15 | Break |
| 2:15-2:45 | Ideation session |
| 2:45-3:15 | Rapid prototyping |
| 3:15-3:45 | Testing and feedback |
| 3:45-4:00 | Reflection and close |

### Format 2: One-Day Intensive (8 hours)

| Time | Activity |
|------|----------|
| 9:00-9:30 | Welcome, overview |
| 9:30-10:30 | Empathy deep dive |
| 10:30-10:45 | Break |
| 10:45-12:00 | Define phase |
| 12:00-13:00 | Lunch |
| 13:00-14:00 | Ideation session |
| 14:00-15:00 | Prototyping |
| 15:00-15:15 | Break |
| 15:15-16:00 | Testing |
| 16:00-16:45 | Presentations |
| 16:45-17:00 | Reflection and close |

### Format 3: Design Sprint (5 days)

| Day | Focus | Outcomes |
|-----|-------|----------|
| Monday | Empathize | User research complete |
| Tuesday | Define | Problem framed, HMW set |
| Wednesday | Ideate | Solutions generated |
| Thursday | Prototype | Testable prototype built |
| Friday | Test | User feedback gathered |

| Ideate | List brainstorming rules; describe ideation methods |
| Prototype | Explain prototype purposes; describe fidelity levels |

|--------|------------|

| Define | Synthesize insights; write HMW statements |
| Ideate | Facilitate brainstorming; apply SCAMPER |

| Module | Objectives |
|--------|------------|

## Facilitation Guidelines

### Before the Session

1. **Prepare materials**: Sticky notes, markers, paper, templates
2. **Set up space**: Movable furniture, wall space, breakout areas
3. **Define challenge**: Have a real problem to work on
4. **Brief participants**: Send pre-work if needed

### During the Session

| Do | Don't |
|----|-------|
| Model the mindsets | Lecture for too long |
| Encourage wild ideas | Judge ideas early |
| Time-box activities | Let discussions drift |
| Keep energy high | Skip breaks |
| Document visually | Rely only on verbal |
| Celebrate failures | Punish mistakes |

### Common Challenges

| Challenge | Solution |
|-----------|----------|
| Participants stuck | Provide constraints, examples |
| Dominant voices | Use silent brainstorming first |
| Resistance to ambiguity | Reassure, explain iteration |
| Shallow empathy | Push for specifics, "tell me more" |
| Jumping to solutions | Redirect to problem space |

### Post-Session

1. **Document outputs**: Photos, summaries
2. **Gather feedback**: What worked, what to improve
3. **Share resources**: Templates, readings, next steps
4. **Follow up**: Check on application, offer support
