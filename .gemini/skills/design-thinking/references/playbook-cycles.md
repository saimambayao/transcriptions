## Divergent vs Convergent Thinking

The core mental states in design thinking.

### Double Diamond Visualization

```
    PROBLEM SPACE              SOLUTION SPACE
         ↓                          ↓
    DIVERGE                    DIVERGE
      /   \                      /   \
     /     \                    /     \
    /       \                  /       \
   /         \                /         \
   \         /                \         /
    \       /                  \       /
     \     /                    \     /
      \   /                      \   /
    CONVERGE                   CONVERGE
```

### Thinking Mode Comparison

| Divergent Thinking | Convergent Thinking |
|-------------------|---------------------|
| New problem statements | Point of decision |
| Discuss possibilities | Identify actions |
| Identify options | Prioritize |
| Different perspectives | Select |
| **Differentiate** | **Integrate** |
| Expand creative framework | Refine and consolidate |

### When to Transition

- **Time pressure**: Move deadlines forward to force mindset change
- **Sufficient ideas**: When quantity is adequate for selection
- **External validation**: Present to other groups for objective selection
- **Defined functionalities**: Match ideas against established criteria

## The Groan Zone

The challenging transition from divergent to convergent thinking.

### What Is the Groan Zone?

The uncomfortable period when:
- Ideas feel incomplete
- Team is uncertain about direction
- Pressure to decide conflicts with desire to explore
- "Mental leap" required from possibilities to solutions

### Steve Jobs and the Reality Distortion Field

Steve Jobs mastered managing the groan zone by having the instinct to choose the right time for mindset change. The "reality distortion field" (term from Bud Tribble at Apple, referencing Star Trek's "The Menagerie") describes the ability to master this mental leap.

### Navigating the Groan Zone

**Techniques**:
1. Set clear time constraints
2. Present remaining ideas to outside groups
3. Use voting (Post-it dots) for democratic selection
4. Keep eliminated ideas as knowledge reservoir
5. Recognize when to stop diverging

**Warning**: Don't fall in love with specific ideas - this blocks objective selection.

## Micro Cycle

The 6-phase iterative process for each design thinking iteration.

### Phase Overview

```
Understand → Observe → Define Point of View → Ideate → Prototype → Test [→ Reflect]
```

### Phase Details

#### 1. UNDERSTAND

**Purpose**: Grasp the problem, context, and constraints.

**Key Activities**:
- Define problem at right comfort level
- Expand framework with "WHY?" questions
- Narrow framework with "HOW?" questions
- Answer WH questions: Who, Why, What, When, Where, How

**Example - Further Education**:
```
                         ┌─ WHY? ─┐
                         │ • Achieve individual fulfillment
                         │ • Take care of families
                         │ • Increase economic value
                         │
 EDUCATE OURSELVES ──────┼─────────────────────
 IN DESIGN THINKING      │
                         │
                         └─ HOW? ─┐
                                  │ • Read The Playbook
                                  │ • Learn from others
                                  │ • Archive information
```

#### 2. OBSERVE

**Purpose**: Gather real-world insights about users.

**Key Activities**:
- Document and visualize findings
- Use idea boards, vision boards, photos
- Create mood pictures and life situation photos
- Build empathy through observation

**Documentation Methods**:
- Qualitative observation
- Photo/video documentation
- Mind maps
- Daily story based on photos

#### 3. DEFINE POINT OF VIEW

**Purpose**: Synthesize insights into actionable focus.

**Key Activities**:
- Draw upon and interpret all findings
- Share stories and experiences across team
- Establish common knowledge base
- Develop/revise personas

**POV Formulations**:

| Approach | Template |
|----------|----------|
| **How Might We** | How might we help [user] achieve [goal]? |
| **Stanford POV** | [User] needs [need] because [surprising insight] |
| **User Story** | As a [role], I want [action] to achieve [benefit] |

#### 4. IDEATE

**Purpose**: Generate diverse solution concepts.

**Key Activities**:
- Brainstorming
- Sketching concepts
- Apply creativity techniques
- Develop many different concepts

**Targeted Brainstorming Options**:
- Critical functions brainstorming
- Benchmarking with other industries
- Dark horse (deliberately ignore constraints)
- Funky prototype (ignore all limits)

#### 5. PROTOTYPE

**Purpose**: Make ideas tangible for testing.

**Key Activities**:
- Build physical or digital representations
- Test with potential users
- Get feedback for improvement

**Motto**: Love it, change it, or leave it.

#### 6. TEST

**Purpose**: Learn from user interaction.

**Key Activities**:
- Test with colleagues (initial)
- Test with potential users (essential)
- Use digital testing tools
- Collect qualitative feedback

#### 7. REFLECT (Optional/Wrap-up)

**Purpose**: Review and improve process.

**Key Activities**:
- Review chosen direction
- Check social acceptability
- Check resource efficiency
- Update personas based on findings

**Retrospective Board**:
- Things we continue to do
- Things we will not continue to do
- Things we want to try out
- Things that are irrelevant

## Macro Cycle

The 7-step journey from problem to implemented solution.

### Visual Overview

```
UNCERTAINTY/CREATIVITY ─────────────────────────────────────→ CERTAINTY/INNOVATION
        ↑                                                              ↑
        │    (1)    (2)    (3)    (4)    (5)                          │
        │     ↻      ↻      ↻      ↻      ↻                           │
        │                                  ↓                          │
        │                              [GROAN ZONE]                   │
        │                                  ↓                          │
        │                                 (6)                         │
        │                                  ↓                          │
        │                        (7) A → B → C → D                    │
        │                                                             │
PROBLEM ─────────────────────────────────────────────────→ SOLUTION
                              TIME
```

### Step-by-Step Process

#### DIVERGENT PHASE (Steps 1-5)

##### Step 1: Initial Ideas Brainstorming

**Purpose**: Get all ideas out, establish common understanding.

**Time**: 20 minutes

**Key Questions**:
- Which ideas come to mind spontaneously?
- Which solution approaches do others pursue?
- What can we do differently?
- Do we all understand the problem the same way?

**Output**: Quantity over quality - every idea on a Post-it, spoken aloud.

---

##### Step 2: Critical Functionalities

**Purpose**: Identify essential user experiences.

**Time**: 1-2 hours (depending on complexity)

**Key Questions**:
- Which functionalities are mandatory?
- What experience is absolutely necessary for the user?
- What is the relationship between function and experience?

**Output**: 10-20 critical functions drafted, built, and tested.

---

##### Step 3: Benchmarking Ideas

**Purpose**: Break free from original solution concepts.

**Time**: 30 min brainstorming + 30 min benchmarks + 30 min clustering

**Key Questions**:
- Which successful concepts from other industries apply?
- Which experiences illuminate the problem differently?
- What is the relation between problem and other experiences?

**Process**:
1. Brainstorm ideas relating to problem
2. Brainstorm industries/experiences
3. Identify three best from each
4. Combine and develop 2-3 further

---

##### Step 4: Dark Horse Prototype

**Purpose**: Boost creativity by removing constraints.

**Time**: 50 minutes

**Key Questions**:
- Which radical possibilities have not been considered?
- Which experiences lie outside anything imaginable?
- Are there products/services that would expand value creation?

**Technique**: Omit essential elements of given situation:
- "IT service desk without IT problems?"
- "Windshield wiper without windshield?"
- "Cemetery if no one died?"

**Mindset**: Maximum risk, leave comfort zone, "do it anyway."

---

##### Step 5: Funky Prototype

**Purpose**: Maximize learning while minimizing cost.

**Time**: 1 hour

**Key Questions**:
- What crazy ideas are super cool?
- For which idea would you have to ask forgiveness?
- What does an ad hoc, unplanned idea look like?

**Approach**: Remove ALL budget restrictions and cost considerations. Focus purely on benefit.

---

#### THE GROAN ZONE (Step 6)

##### Step 6: Vision Prototype

**Purpose**: Transition from divergent to convergent phase.

**Time**: ~2 hours + testing with 3+ users

**Combines**:
- Prior knowledge (use with caution)
- Best initial ideas
- Most important critical functionalities
- New ideas from other industries
- Initial user experience
- Insights from dark horse
- Simplest possible solution

**Key Questions**:
- Does vision generate enough attention that user absolutely wants it?
- Does vision give leeway for user's dreams?
- Is value offer convincing?
- What else would make experience perfect?

**Best Practice**: Involve lead users who are highly motivated to satisfy their needs.

---

#### CONVERGENT PHASE (Step 7)

##### Step 7: Concretize the Vision

**Purpose**: Iteratively detail and elaborate the selected idea.

**Sub-Steps**:

| Stage | Focus | Description |
|-------|-------|-------------|
| **A. Functional Prototype** | Critical variables | MVP testing under real conditions |
| **B. Finished Prototype** | Multiple functions | Reality yields truth |
| **C. Final Prototype** | Elegant realization | Simple functionality, convincing solution |
| **D. Implementation Plan** | "How to bring it home" | Supplier/partner involvement |

## Integration with Scrum

Design thinking phases map to agile methodologies.

### ISO 13407 Mapping

| ISO Phase | Design Thinking Equivalent |
|-----------|---------------------------|
| Planning, process | Understand |
| Analysis, use context | Observe |
| Specifications, user requirements | Define POV |
| Prototype (design variants) | Prototype |
| Evaluation | Test |

### Scrum Integration

```
Product Backlog → Sprint Backlog → SPRINT (1-4 weeks) → Retrospective → Product
                                      ↓
                              [User stories tested
                               during sprint]
```

### Simplified Cycles

**Swisscom (3-Phase)**:
- **Hear**: Understand project, customer problems, gather experience
- **Create**: Transform learnings into solutions, generate possibilities
- **Deliver**: Concretize ideas, create/test prototypes, gain insights

**Japanese Model (4-Phase)**:
- **Empathy** → **Analysis** → **Prototype** → **Co-Creation**
