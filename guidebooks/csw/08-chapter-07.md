# Chapter 7 — Step 4: Explore Options (E)

You defined the problem. You gathered evidence. Now you must generate solutions, evaluate them against consistent criteria, and recommend the best one. This is the step where analysis becomes advice.

Decision-makers do not want a single option presented as the only way forward. They want to see that you considered alternatives, weighed trade-offs, and arrived at a recommendation through systematic evaluation. One option is a directive. Three options are a decision.

## 7.1 Generating Options (Minimum 2-3)

### How to Generate Options

**Start broad, then narrow.** Your initial list should include 4-6 possibilities. After evaluation, you will present the top 2-3 to the decision-maker.

**Methods for generating options:**

| Method | How It Works | Best For |
|--------|-------------|----------|
| **Brainstorming** | Generate as many ideas as possible without filtering. Quantity first, quality later. | Creative or novel problems |
| **Benchmarking** | Examine how other organizations solved similar problems. Adapt their approaches. | Problems with known solutions elsewhere |
| **Incremental vs. transformational** | Generate one option that modifies the current approach and another that replaces it entirely. | Situations where the status quo has some value |
| **Stakeholder-driven** | Ask each stakeholder group what solution they prefer. Map the range of preferences. | Politically sensitive issues |
| **Resource-based** | Design options around different resource levels: minimal, moderate, maximum investment. | Budget-constrained environments |

### Rules for Option Generation

1. **Minimum two options, target three.** One option is not a choice. Two options create a binary. Three options give the decision-maker genuine discretion.
2. **No strawmen.** Every option must be genuinely viable. Do not include an obviously bad option to make your preferred option look good. Decision-makers see through this.
3. **Include the status quo.** If doing nothing is a real possibility, present it as Option A. Show its consequences alongside the action options.
4. **Vary the approach, not just the scale.** "Build 100 schools" and "Build 200 schools" are not two options. They are one option at two scales. Real alternatives differ in approach: "Build schools" vs. "Subsidize private providers" vs. "Mobile learning units."

> **TIP:** If you cannot generate at least two genuinely different options, you have not researched deeply enough. Return to Step 3 and look for alternative approaches, different models, or different intervention points.

## 7.2 Evaluation Criteria

Evaluate every option against the same five criteria. This ensures consistency and transparency.

### 7.2.1 Feasibility

Can you actually do this?

| Feasibility Dimension | Questions to Ask |
|----------------------|-----------------|
| **Technical** | Does BARMM have the technical capacity to implement? |
| **Administrative** | Are the institutional mechanisms in place? |
| **Political** | Is there sufficient political support? Will Parliament or the executive approve? |
| **Legal** | Is the option authorized under the BOL, BAAs, and applicable laws? |
| **Timeline** | Can it be implemented within the required timeframe? |

### 7.2.2 Effectiveness

Does it solve the problem?

- How much of the problem does this option address?
- Does it target root causes or symptoms?
- What is the projected impact on the measurable indicators in your problem statement?
- How confident are you in the projected outcome?

### 7.2.3 Efficiency

Does it make good use of resources?

- What is the total cost (direct and indirect)?
- What is the cost per beneficiary or per unit of outcome?
- Does it leverage existing infrastructure and personnel, or does it require new investment?
- Are there economies of scale?

### 7.2.4 Sustainability

Will the results last?

- Can the option be maintained after initial funding ends?
- Does it build institutional capacity or create dependency on external support?
- Is it resilient to political changes, leadership transitions, and budget fluctuations?
- Does it create self-reinforcing mechanisms?

### 7.2.5 Alignment with Organizational Goals

Does it fit BARMM's direction?

- How does this option advance the BDP 2023-2028?
- Does it align with the 12-point priority agenda?
- Is it consistent with the BOL's vision for the Bangsamoro?
- Does it support moral governance principles?

## 7.3 Impact Analysis

For each option, assess the impact on different stakeholder groups and dimensions.

**Impact Analysis Matrix:**

| Impact Dimension | Option A | Option B | Option C |
|-----------------|----------|----------|----------|
| **Beneficiaries** | Who benefits? How many? How much? | | |
| **Affected communities** | Who is disadvantaged? What mitigation exists? | | |
| **Budget** | Total cost, funding source, fiscal sustainability | | |
| **Personnel** | Staffing requirements, training needs | | |
| **Legal/compliance** | Regulatory requirements, COA exposure | | |
| **Political** | Level of support, potential opposition | | |
| **Timeline** | Implementation duration, milestones | | |
| **Risk** | Key risks and probability of occurrence | | |

> **BANGSAMORO NOTE:** Impact analysis in BARMM should always include assessment of effects on **peace and security**, **indigenous peoples** (where applicable), **gender equity**, and **Shari'ah compliance** (where applicable). These are not optional add-ons. The BOL and BDP require attention to these dimensions.

## 7.4 Cost-Benefit Analysis

For options with significant financial implications, conduct a structured CBA.

### Basic CBA Framework for CSW

| Element | Option A | Option B | Option C |
|---------|----------|----------|----------|
| **Direct costs** | | | |
| Personnel Services (PS) | | | |
| Maintenance & Other Operating Expenses (MOOE) | | | |
| Capital Outlay (CO) | | | |
| **Total cost** | | | |
| **Quantifiable benefits** | | | |
| Revenue generated or saved | | | |
| Service delivery improvement (quantified) | | | |
| Efficiency gains (time, resources) | | | |
| **Non-quantifiable benefits** | | | |
| Institutional capacity built | | | |
| Public trust strengthened | | | |
| BDP alignment achieved | | | |
| **Benefit-Cost Ratio** | Benefits / Costs | | |
| **Payback Period** | When benefits exceed costs | | |

> **CAUTION:** Do not fabricate precision. If you cannot reliably estimate a benefit, state it qualitatively rather than assigning a made-up number. "Improved public trust" is an honest non-quantifiable benefit. "PHP 50 million in public trust value" is fiction.

### When CBA Is Not Appropriate

Not every CSW requires formal CBA. Cost-benefit analysis works best for:

- Capital projects and infrastructure
- Program expansions with clear budget implications
- Options that differ significantly in cost

CBA is less useful for:

- Legal compliance questions (you must comply regardless of cost)
- Ethical or human rights issues
- Situations where benefits are primarily intangible

In these cases, use the five evaluation criteria (Section 7.2) without forcing a numerical CBA.

## 7.5 Prioritizing and Ranking Options

Use an **Options Evaluation Matrix** to score each option against the five criteria.

### Options Evaluation Matrix

| Criterion | Weight | Option A | Option B | Option C |
|-----------|--------|----------|----------|----------|
| Feasibility | 25% | | | |
| Effectiveness | 30% | | | |
| Efficiency | 15% | | | |
| Sustainability | 15% | | | |
| Alignment | 15% | | | |
| **Weighted Total** | 100% | | | |
| **Rank** | | | | |

**Scoring scale:** 1 (Very Low) to 5 (Very High)

**Weighted score** = Score x Weight. Sum weighted scores for each option. Highest total is the top-ranked option.

**Assigning weights:** The default weights above work for most CSW. Adjust based on the decision-maker's priorities. If the Chief Minister has emphasized fiscal discipline, increase the Efficiency weight. If the issue is a legal compliance matter, increase Feasibility.

**Example — ECCD Teacher Shortage options:**

| Criterion | Weight | A: Salary Supplement | B: Bridging Program | C: Combined (A+B) |
|-----------|--------|---------------------|--------------------|--------------------|
| Feasibility | 25% | 3 (requires BAA) | 4 (admin order) | 3 |
| Effectiveness | 30% | 4 (addresses root cause) | 3 (addresses one factor) | 5 (addresses both) |
| Efficiency | 15% | 3 (recurring cost) | 4 (one-time + recurring) | 3 |
| Sustainability | 15% | 4 (if legislated) | 3 (needs annual funding) | 4 |
| Alignment | 15% | 5 (BDP Goal 2) | 4 (BDP Goal 2) | 5 |
| **Weighted Total** | 100% | **3.7** | **3.6** | **4.2** |
| **Rank** | | 2 | 3 | **1** |

## 7.6 Validating with Stakeholders

Before finalizing your recommendation, test your top option(s) with key stakeholders.

**What to validate:**

| Question | Stakeholder |
|----------|-----------|
| Is this option technically feasible? | Subject matter experts, implementing agency |
| Will this option have unintended consequences? | Affected communities, frontline staff |
| Can we fund this? | MFBM, budget officer |
| Will this pass Parliament (if legislative action is needed)? | Committee staff, political advisors |
| Does this comply with applicable laws? | Legal counsel |

Stakeholder validation is not asking for permission. It is a reality check. If a key stakeholder identifies a fatal flaw in your top option, better to find out now than after the decision-maker approves it.

## 7.7 Preparing for Recommendation

Your Step 4 output feeds directly into Step 5 (Submit). Before moving forward, confirm:

- [ ] At least 2-3 genuine options were generated (no strawmen)
- [ ] Each option was evaluated against all five criteria
- [ ] Impact analysis covered all relevant dimensions
- [ ] CBA was conducted where appropriate
- [ ] Options are ranked with a clear top recommendation
- [ ] Key stakeholders validated the top option
- [ ] The rationale for the recommendation is documented with evidence

**Prepare a clear recommendation statement:**

> Based on the evaluation of three options against five criteria (feasibility, effectiveness, efficiency, sustainability, and alignment), **Option C (Combined Salary Supplement and Bridging Program)** is recommended. It scored highest overall (4.2 out of 5.0), primarily due to its effectiveness in addressing both root causes of the ECCD teacher shortage (salary competitiveness and qualification gaps). The estimated total cost is PHP 340 million over three years, funded from the BARMM block grant education allocation. Implementation requires a new BAA authorizing salary supplements and an administrative order establishing the bridging program.

## 7.8 Why This Step Matters — Consequences of Skipping

When you skip option exploration:

**You default to the first idea.** Without generating alternatives, you recommend whatever occurred to you first. First ideas are rarely the best ideas. They reflect your biases, not the evidence.

**The decision-maker has no choice.** A single option forces a yes-or-no decision. The decision-maker cannot weigh trade-offs, adjust scope, or combine elements from different approaches.

**You miss superior solutions.** Brainstorming, benchmarking, and stakeholder input often produce options you would never have conceived alone. Skipping this step means missing them.

**Your recommendation lacks justification.** "We recommend this because it is the best option" means nothing if you have not shown what the other options were and why they ranked lower.

**Real BARMM example of skipping Step 4:**

A ministry prepared a project proposal for a regional skills training center. The proposal presented one option: construct a new training center in Cotabato City at a cost of PHP 850 million. MFBM rejected the proposal, noting that the ministry had not explored alternatives.

When forced to explore options, the ministry discovered three alternatives: (A) renovating an existing underutilized government building (PHP 120 million), (B) partnering with private TESDA-accredited institutions through a subsidy model (PHP 45 million per year), and (C) a hybrid approach combining a smaller new facility with private partnerships (PHP 280 million plus PHP 20 million per year).

Option C was eventually approved. It cost 67% less than the original proposal and achieved broader geographic coverage through the private partnership component. The initial single-option approach would have either been rejected entirely or approved at a cost three times higher than necessary.

---

**Chapter Summary**

- Generate at least 2-3 genuine options. No strawmen. Include the status quo if relevant.
- Evaluate every option against five criteria: feasibility, effectiveness, efficiency, sustainability, and alignment.
- Conduct impact analysis across stakeholder groups and key dimensions.
- Perform cost-benefit analysis for options with significant financial implications.
- Rank options using a weighted evaluation matrix. Document the scoring and rationale.
- Validate the top option with key stakeholders before finalizing.
- Prepare a clear recommendation statement with evidence-based justification.
- Skipping this step leads to first-idea defaults, unjustified recommendations, and missed superior solutions.
